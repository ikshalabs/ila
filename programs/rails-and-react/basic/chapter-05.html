Chapter 05 · Phase 3: React Fundamentals

React Components and JSX

Learn to build user interfaces with React's component model. Understand JSX syntax, functional components, props for data flow, conditional rendering, list rendering, and component composition patterns that scale from simple pages to complex applications.

Estimated: 3–4 days
3 Advancement Levels
3 Assignments

Why This Matters

React is the most widely used JavaScript library for building user interfaces. It introduces a fundamental shift in how you think about UI: instead of manipulating the DOM directly, you describe what the interface should look like for a given state, and React figures out how to update the DOM efficiently.

Components are React's building blocks. Every button, form, card, page, and layout in a React application is a component. They are reusable, composable, and testable. Once you understand components and props, you understand how to build any UI — because every complex interface is just a tree of simple components.

Key Insight

React's mental model is: UI = f(state). Your interface is a function of your data. When the data changes, React re-renders only the parts that need updating. This is why React applications feel fast — you never manually manipulate DOM nodes.

Key Concepts

1. JSX — JavaScript Meets HTML

JSX is a syntax extension that lets you write HTML-like code inside JavaScript. It is not a template language — it compiles to regular JavaScript function calls. Every React component returns JSX that describes what should appear on screen.

jsx
// JSX looks like HTML but it is JavaScript
const greeting = <h1>Hello, World!</h1>;

// Embedding expressions with curly braces
const name = "Alice";
const element = <h1>Hello, {name}!</h1>;

// JSX expressions can be any valid JavaScript
const price = <span>${(29.99 * 1.1).toFixed(2)}</span>;

// Attributes use camelCase (className, not class; onClick, not onclick)
const button = (
  <button className="btn-primary" onClick={handleClick}>
    Click Me
  </button>
);

// Multi-line JSX must be wrapped in parentheses
// and have a single root element (or use fragments)
const card = (
  <>
    <h2>Title</h2>
    <p>Description</p>
  </>
);
JSX is Not HTML

Key differences: use className instead of class, htmlFor instead of for, and style takes an object (style={{ color: "red" }}), not a string. Self-closing tags must have a slash: <img />, <br />, <input />.

2. Functional Components

Components are JavaScript functions that return JSX. They accept a single argument called props (properties) — an object containing data passed from the parent component. By convention, component names start with a capital letter.

jsx
// A simple component — receives props, returns JSX
function UserCard({ name, email, role }) {
  return (
    <div className="user-card">
      <h3>{name}</h3>
      <p>{email}</p>
      <span className="badge">{role}</span>
    </div>
  );
}

// Arrow function syntax (equally valid)
const Badge = ({ text, variant = "default" }) => (
  <span className={`badge badge-${variant}`}>{text}</span>
);

// Using components — pass data via props
function App() {
  return (
    <div>
      <UserCard name="Alice" email="alice@example.com" role="admin" />
      <UserCard name="Bob" email="bob@example.com" role="user" />
    </div>
  );
}

3. Props — Data Flow

Props flow one direction: from parent to child. A child component never modifies its props. This unidirectional data flow makes React applications predictable and debuggable. You always know where data comes from by looking up the component tree.

jsx
// Props can be any JavaScript value
function ProductCard({ product, onAddToCart }) {
  return (
    <div className="product-card">
      <img src={product.imageUrl} alt={product.name} />
      <h3>{product.name}</h3>
      <p className="price">${product.price.toFixed(2)}</p>
      <button onClick={() => onAddToCart(product.id)}>
        Add to Cart
      </button>
    </div>
  );
}

// The children prop — content between opening and closing tags
function Card({ title, children }) {
  return (
    <div className="card">
      <div className="card-header">{title}</div>
      <div className="card-body">{children}</div>
    </div>
  );
}

// Usage — children is everything between the tags
<Card title="User Profile">
  <p>Name: Alice</p>
  <p>Email: alice@example.com</p>
</Card>

4. Conditional Rendering

Since JSX is JavaScript, you use JavaScript expressions for conditional rendering. The three most common patterns are ternary operators, logical AND (&&), and early returns.

jsx
function UserGreeting({ user, isLoading, error }) {
  // Early return pattern — handle edge cases first
  if (isLoading) return <Spinner />;
  if (error) return <ErrorMessage message={error} />;
  if (!user) return <p>No user found.</p>;

  return (
    <div>
      <h2>Welcome, {user.name}!</h2>

      {/* Ternary — choose between two elements */}
      {user.isAdmin ? (
        <AdminDashboard />
      ) : (
        <UserDashboard />
      )}

      {/* Logical AND — show only if condition is true */}
      {user.unreadCount > 0 && (
        <Badge text={`${user.unreadCount} new`} />
      )}
    </div>
  );
}

5. Rendering Lists

Use JavaScript's map() to transform arrays of data into arrays of JSX elements. Every element in a list must have a unique key prop so React can efficiently track which items change.

jsx
function ArticleList({ articles }) {
  if (articles.length === 0) {
    return <p className="empty-state">No articles yet.</p>;
  }

  return (
    <ul className="article-list">
      {articles.map((article) => (
        <li key={article.id}>
          <ArticleCard
            title={article.title}
            summary={article.summary}
            author={article.user.name}
            createdAt={article.created_at}
          />
        </li>
      ))}
    </ul>
  );
}
Never Use Array Index as Key

Using key={index} causes bugs when the list is reordered, filtered, or items are inserted. Always use a stable, unique identifier like id from the database. If your data does not have an ID, that is a data modeling problem to fix first.

6. Component Composition

Composition is how you build complex UIs from simple components. Instead of inheritance, React uses composition — components contain other components, pass them data through props, and use the children prop for flexible layouts.

jsx
// Layout component — wraps any content
function PageLayout({ title, children }) {
  return (
    <div className="page">
      <header className="page-header">
        <h1>{title}</h1>
      </header>
      <main className="page-content">
        {children}
      </main>
      <Footer />
    </div>
  );
}

// Composing a full page from small components
function ArticlesPage() {
  return (
    <PageLayout title="All Articles">
      <SearchBar placeholder="Search articles..." />
      <ArticleList articles={articles} />
      <Pagination total={totalPages} current={page} />
    </PageLayout>
  );
}
Pro Tip

A good component does one thing well. If a component is doing too much (fetching data, handling forms, rendering lists, AND managing layout), split it into smaller components. Each component should be easy to describe in one sentence.

Self-Check

Can you build a component from scratch in under 5 minutes?

Time yourself building a NotificationBanner component that accepts type (success, warning, error), message, and an optional onDismiss callback. If it takes longer than 5 minutes, practice the component creation workflow more.

Do you understand the difference between props and children?

Props are named attributes (<Card title="Hello" />). Children is the content between tags (<Card>content here</Card>). Use props for specific data, children for flexible content that the parent decides.

Can you explain why keys matter in lists?

Keys tell React which items changed, were added, or were removed. Without stable keys, React cannot efficiently update the DOM and may lose input state or cause visual glitches when lists change.
