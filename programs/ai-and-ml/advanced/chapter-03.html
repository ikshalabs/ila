Generative Models

From Variational Autoencoders to Diffusion Models, learn the theory and implementation of systems that create realistic images, audio, and video. Understand the training dynamics, failure modes, and evaluation metrics that separate toy demos from production-quality generation.

Why This Matters

Generative AI has moved from a research curiosity to a multi-billion dollar industry in under five years. Models that generate photorealistic images (Stable Diffusion, DALL-E), coherent long-form text (GPT-4), and realistic video (Sora) are reshaping creative industries, software development, and scientific research. Understanding how these models work — not just what they produce — is essential for building, evaluating, and deploying generative systems responsibly.

This chapter covers the three dominant paradigms: Variational Autoencoders (principled probabilistic generation), GANs (adversarial training for sharp outputs), and Diffusion Models (iterative denoising for state-of-the-art quality). Each has distinct trade-offs in training stability, output quality, diversity, and computational cost.

Key Insight

All generative models solve the same fundamental problem: learning to sample from an unknown data distribution p(x). They differ in how they model that distribution — explicit density estimation (VAEs), implicit density via adversarial training (GANs), or score-based iterative refinement (diffusion). Your choice depends on whether you need diversity, quality, controllability, or training stability.

Key Concepts

1. Variational Autoencoders (VAEs)

A VAE learns a compressed latent representation of data by training an encoder (maps input to a distribution in latent space) and a decoder (maps latent samples back to data space). The key innovation is the reparameterization trick, which allows backpropagation through the stochastic sampling step.

The loss function combines reconstruction quality (how well the decoder recreates the input) with a KL divergence term (how close the latent distribution is to a standard Gaussian). This regularization ensures the latent space is smooth and interpolable. 

  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
  │  Input x │──▶│ Encoder  │──▶│  z ~ N(  │──▶│ Decoder  │──▶ x_hat
  │          │    │ q(z|x)   │    │  mu, sig)│    │ p(x|z)   │
  └──────────┘    └──────────┘    └──────────┘    └──────────┘
                        │                               │
                        │       KL Divergence            │  Reconstruction
                        └───────────┬───────────────────┘  Loss
                                    │
                              Total Loss = Recon + beta * KL

  Reparameterization trick: z = mu + sigma * epsilon,  epsilon ~ N(0, I)

python
import torch
import torch.nn as nn
import torch.nn.functional as F

class VAE(nn.Module):
    """Variational Autoencoder for image generation."""

    def __init__(self, latent_dim=128, img_channels=3):
        super().__init__()
        self.latent_dim = latent_dim

        # Encoder: image -> (mu, log_var)
        self.encoder = nn.Sequential(
            nn.Conv2d(img_channels, 32, 4, stride=2, padding=1),
            nn.ReLU(),
            nn.Conv2d(32, 64, 4, stride=2, padding=1),
            nn.ReLU(),
            nn.Conv2d(64, 128, 4, stride=2, padding=1),
            nn.ReLU(),
            nn.Flatten(),
        )
        self.fc_mu = nn.Linear(128 * 4 * 4, latent_dim)
        self.fc_var = nn.Linear(128 * 4 * 4, latent_dim)

        # Decoder: z -> image
        self.fc_decode = nn.Linear(latent_dim, 128 * 4 * 4)
        self.decoder = nn.Sequential(
            nn.ConvTranspose2d(128, 64, 4, stride=2, padding=1),
            nn.ReLU(),
            nn.ConvTranspose2d(64, 32, 4, stride=2, padding=1),
            nn.ReLU(),
            nn.ConvTranspose2d(32, img_channels, 4, stride=2, padding=1),
            nn.Sigmoid(),
        )

    def reparameterize(self, mu, log_var):
        """Sample z using the reparameterization trick."""
        std = torch.exp(0.5 * log_var)
        eps = torch.randn_like(std)
        return mu + eps * std

    def forward(self, x):
        h = self.encoder(x)
        mu, log_var = self.fc_mu(h), self.fc_var(h)
        z = self.reparameterize(mu, log_var)
        decoded = self.fc_decode(z).view(-1, 128, 4, 4)
        recon = self.decoder(decoded)
        return recon, mu, log_var

    def loss_function(self, recon_x, x, mu, log_var, beta=1.0):
        recon_loss = F.mse_loss(recon_x, x, reduction='sum')
        kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())
        return recon_loss + beta * kl_loss

2. GANs — Architecture and Training

Generative Adversarial Networks pit two networks against each other: a generator that creates fake samples and a discriminator that tries to distinguish real from fake. The generator improves by learning to fool the discriminator, while the discriminator improves by getting better at detection. This adversarial dynamic produces strikingly sharp outputs.

Training GANs is notoriously difficult. Common failure modes include mode collapse (generator produces only a few distinct outputs), training instability (oscillating losses), and vanishing gradients (discriminator becomes too strong). Techniques like spectral normalization, progressive growing, and Wasserstein loss have made training more stable.

python
import torch
import torch.nn as nn

class Generator(nn.Module):
    """DCGAN-style generator: latent vector -> image."""

    def __init__(self, latent_dim=100, img_channels=3):
        super().__init__()
        self.net = nn.Sequential(
            # latent_dim -> 512 x 4 x 4
            nn.ConvTranspose2d(latent_dim, 512, 4, 1, 0, bias=False),
            nn.BatchNorm2d(512), nn.ReLU(True),
            # 512 x 4 x 4 -> 256 x 8 x 8
            nn.ConvTranspose2d(512, 256, 4, 2, 1, bias=False),
            nn.BatchNorm2d(256), nn.ReLU(True),
            # 256 x 8 x 8 -> 128 x 16 x 16
            nn.ConvTranspose2d(256, 128, 4, 2, 1, bias=False),
            nn.BatchNorm2d(128), nn.ReLU(True),
            # 128 x 16 x 16 -> 3 x 32 x 32
            nn.ConvTranspose2d(128, img_channels, 4, 2, 1, bias=False),
            nn.Tanh(),
        )

    def forward(self, z):
        return self.net(z.view(z.size(0), -1, 1, 1))


class Discriminator(nn.Module):
    """DCGAN-style discriminator with spectral normalization."""

    def __init__(self, img_channels=3):
        super().__init__()
        self.net = nn.Sequential(
            nn.utils.spectral_norm(nn.Conv2d(img_channels, 128, 4, 2, 1)),
            nn.LeakyReLU(0.2),
            nn.utils.spectral_norm(nn.Conv2d(128, 256, 4, 2, 1)),
            nn.LeakyReLU(0.2),
            nn.utils.spectral_norm(nn.Conv2d(256, 512, 4, 2, 1)),
            nn.LeakyReLU(0.2),
            nn.Conv2d(512, 1, 4, 1, 0),
        )

    def forward(self, x):
        return self.net(x).view(-1)

3. Diffusion Models

Diffusion models generate data by learning to reverse a noise process. The forward process gradually adds Gaussian noise to data over T timesteps until the signal is completely destroyed. The reverse process trains a neural network to predict and remove the noise at each step, starting from pure noise and iteratively refining toward a clean sample.

This approach produces state-of-the-art image quality and diversity, surpassing GANs on most benchmarks. The key breakthrough was realizing that the denoising network can be trained with a simple MSE loss to predict the noise added at each step, making training far more stable than GANs.

python
import torch
import torch.nn as nn

class SimpleDiffusion(nn.Module):
    """Simplified DDPM (Denoising Diffusion Probabilistic Model)."""

    def __init__(self, denoise_model, T=1000, beta_start=1e-4, beta_end=0.02):
        super().__init__()
        self.model = denoise_model
        self.T = T

        # Linear noise schedule
        betas = torch.linspace(beta_start, beta_end, T)
        alphas = 1.0 - betas
        alphas_cumprod = torch.cumprod(alphas, dim=0)

        self.register_buffer('betas', betas)
        self.register_buffer('sqrt_alphas_cumprod', torch.sqrt(alphas_cumprod))
        self.register_buffer('sqrt_one_minus_alphas_cumprod',
                             torch.sqrt(1.0 - alphas_cumprod))

    def forward_process(self, x0, t, noise=None):
        """Add noise to x0 at timestep t."""
        if noise is None:
            noise = torch.randn_like(x0)

        sqrt_alpha = self.sqrt_alphas_cumprod[t].view(-1, 1, 1, 1)
        sqrt_one_minus = self.sqrt_one_minus_alphas_cumprod[t].view(-1, 1, 1, 1)
        return sqrt_alpha * x0 + sqrt_one_minus * noise

    def training_loss(self, x0):
        """Simple noise prediction loss."""
        t = torch.randint(0, self.T, (x0.size(0),), device=x0.device)
        noise = torch.randn_like(x0)
        x_noisy = self.forward_process(x0, t, noise)
        predicted_noise = self.model(x_noisy, t)
        return nn.functional.mse_loss(predicted_noise, noise)

    @torch.no_grad()
    def sample(self, shape, device):
        """Generate samples via iterative denoising."""
        x = torch.randn(shape, device=device)
        for t in reversed(range(self.T)):
            t_batch = torch.full((shape[0],), t, device=device, dtype=torch.long)
            predicted_noise = self.model(x, t_batch)
            x = self.denoise_step(x, predicted_noise, t)
        return x

4. Image Generation Pipelines

Modern image generation systems like Stable Diffusion operate in a latent space rather than pixel space. An image encoder compresses the input into a lower-dimensional latent representation, the diffusion process operates in this compact space, and a decoder maps the denoised latent back to pixel space. This dramatically reduces computational cost.

python
from diffusers import StableDiffusionPipeline, DPMSolverMultistepScheduler
import torch

# Load Stable Diffusion with an efficient scheduler
pipe = StableDiffusionPipeline.from_pretrained(
    "stabilityai/stable-diffusion-2-1",
    torch_dtype=torch.float16,
)
pipe.scheduler = DPMSolverMultistepScheduler.from_config(pipe.scheduler.config)
pipe = pipe.to("cuda")

# Text-to-image generation
prompt = "A photorealistic landscape of mountains at sunset, 4k, detailed"
negative_prompt = "blurry, low quality, watermark, text"

image = pipe(
    prompt=prompt,
    negative_prompt=negative_prompt,
    num_inference_steps=25,     # DPM-Solver needs fewer steps
    guidance_scale=7.5,          # classifier-free guidance strength
    width=768, height=512,
).images[0]

image.save("generated_landscape.png")

5. Evaluation Metrics (FID, IS, CLIP)

Evaluating generative models is notoriously difficult because "quality" is subjective. The field relies on automated metrics that correlate with human judgment, each measuring a different aspect:

  • FID (Frechet Inception Distance) — measures the distance between feature distributions of real and generated images. Lower is better. The gold standard for image quality + diversity.
  • IS (Inception Score) — measures both quality (confident classifications) and diversity (uniform class distribution). Higher is better, but has known blind spots.
  • CLIP Score — measures alignment between generated images and text prompts using CLIP embeddings. Essential for text-to-image models.
  • LPIPS — measures perceptual similarity between images using deep features. Useful for evaluating reconstruction quality.
python
from torchmetrics.image.fid import FrechetInceptionDistance
from torchmetrics.image.inception import InceptionScore
import torch

# FID: compare real vs generated distributions
fid = FrechetInceptionDistance(feature=2048, normalize=True)

# Add real images (batch of uint8 tensors, shape [N, 3, H, W])
fid.update(real_images, real=True)

# Add generated images
fid.update(generated_images, real=False)

# Compute FID score (lower = better)
fid_score = fid.compute()
print(f"FID: {fid_score:.2f}")
# Good models: FID < 10, State-of-art: FID < 3

# Inception Score
inception = InceptionScore(normalize=True)
inception.update(generated_images)
is_mean, is_std = inception.compute()
print(f"IS: {is_mean:.2f} +/- {is_std:.2f}")

# CLIP Score for text-image alignment
from torchmetrics.multimodal import CLIPScore
clip_metric = CLIPScore(model_name_or_path="openai/clip-vit-base-patch16")
score = clip_metric(generated_images, ["a cat sitting on a chair"] * len(generated_images))
print(f"CLIP Score: {score:.2f}")

6. Ethical Considerations

Generative models raise significant ethical concerns that every practitioner must understand and address:

  • Deepfakes and misinformation — Photorealistic generation enables creation of convincing fake images and videos. Always implement watermarking and provenance tracking.
  • Training data consent — Models trained on web-scraped data may contain copyrighted work or images of identifiable individuals without their consent.
  • Bias amplification — Models reproduce and amplify biases present in training data, including racial, gender, and cultural stereotypes.
  • Environmental cost — Training large generative models requires enormous compute, with significant carbon footprint. Document and minimize this cost.
  • Dual-use risk — The same technology that creates art can create harassment material. Build safety filters and abuse detection into production systems.
Critical Responsibility

If you deploy a generative model, you are responsible for its outputs. Implement content safety classifiers, user reporting mechanisms, and audit logs. "The model did it" is not an acceptable response when harmful content is generated.

What is classifier-free guidance and why does it improve image quality?

Classifier-free guidance trains the diffusion model to denoise both with and without the text condition. At inference, the model's prediction is pushed away from the unconditional output and toward the conditional output by a guidance scale factor. Higher guidance scale produces images more aligned with the prompt but with less diversity. A scale of 7–8 is typical for Stable Diffusion.

Why do VAE outputs look blurry compared to GANs?

VAEs optimize a pixel-level reconstruction loss (MSE or BCE), which penalizes all errors equally. This produces outputs that are the average of all plausible reconstructions, resulting in blurriness. GANs use a discriminator that penalizes unrealistic textures, producing sharper but potentially less diverse outputs. Diffusion models avoid both problems by operating iteratively.

Self-Check

Before moving to Chapter 04, honestly answer these questions.

Can you explain the reparameterization trick and why it is necessary?

Sampling z ~ N(mu, sigma) is not differentiable. The reparameterization trick rewrites z = mu + sigma * epsilon (where epsilon ~ N(0,1)), which moves the stochasticity to a parameter-free noise term. This makes the path from encoder to decoder fully differentiable.

What causes mode collapse in GANs and how do you detect it?

Mode collapse occurs when the generator learns to produce only a few distinct outputs. Detect it by: (1) inspecting generated samples for low diversity, (2) measuring FID (high FID with visually ok samples suggests missing modes), (3) computing the truncation curve. Solutions include Wasserstein loss, spectral normalization, and diversity-promoting regularization.

Why are diffusion models better than GANs for most image generation tasks?

Diffusion models are more stable to train (simple MSE loss), produce higher diversity (no mode collapse), have better FID scores on most benchmarks, and support easier conditioning (classifier-free guidance). GANs are faster at inference (single forward pass vs. many denoising steps) and can be better for real-time applications.

Can you evaluate a generative model beyond just looking at samples?

You should compute FID (quality + diversity), IS (quality + class coverage), CLIP score (text alignment for conditioned models), and LPIPS (perceptual similarity for reconstruction tasks). Always compute on at least 10,000 samples and report confidence intervals.
