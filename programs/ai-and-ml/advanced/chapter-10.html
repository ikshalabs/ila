Capstone: End-to-End ML Platform

Design, build, deploy, and monitor a complete machine learning platform. Integrate every skill from this certification — data pipelines, model training, serving infrastructure, observability, and fairness — into a production-grade system you can defend in a technical review.

Estimated: 5–7 days
3 Advancement Levels
3 Assignments

Why This Matters

Building individual ML components is necessary but not sufficient. The real challenge — and the skill that separates junior practitioners from senior engineers — is integrating those components into a cohesive, reliable platform. A model that works in a notebook means nothing if it cannot be trained reproducibly, deployed safely, monitored continuously, and updated without downtime.

This capstone project synthesizes every chapter in this certification. You will architect a system that ingests raw data, transforms it through feature pipelines, trains and validates models, serves predictions at scale, monitors for drift and degradation, and documents its fairness properties. The result is not a toy — it is a portfolio-grade system that demonstrates production ML engineering competency.

Key Insight

Production ML is 90% engineering and 10% modeling. The capstone forces you to confront the unglamorous but critical parts: configuration management, error handling, graceful degradation, rollback strategies, and operational runbooks. These are the skills hiring managers actually evaluate.

Key Concepts

1. Platform Architecture Design

A well-designed ML platform separates concerns into distinct layers: data ingestion, feature engineering, model training, model registry, serving, and monitoring. Each layer communicates through well-defined interfaces, allowing independent scaling, testing, and replacement. The architecture must support both batch and real-time workloads.

Start by defining your platform configuration as code. Every component, connection string, model hyperparameter, and deployment target should be declarative and version-controlled. This is the foundation that makes everything else reproducible.

@dataclass
class PlatformConfig:
    """Root configuration for the entire ML platform."""
    project_name: str
    environment: Environment
    data_sources: List[DataSourceConfig]
    feature_store: FeatureStoreConfig
    training: TrainingConfig
    serving: ServingConfig
    monitoring: MonitoringConfig
    version: str = "1.0.0"

    @classmethod
    def from_yaml(cls, path: str) -> "PlatformConfig":
        """Load platform configuration from a YAML file."""
        with open(path) as f:
            raw = yaml.safe_load(f)

        env_str = os.getenv("ML_ENVIRONMENT", raw.get("environment", "development"))
        environment = Environment(env_str)

        data_sources = [
            DataSourceConfig(**ds)
            for ds in raw.get("data_sources", [])
        ]

        return cls(
            project_name=raw["project_name"],
            environment=environment,
            data_sources=data_sources,
            feature_store=FeatureStoreConfig(**raw.get("feature_store", {})),
            training=TrainingConfig(**raw.get("training", {})),
            serving=ServingConfig(**raw.get("serving", {})),
            monitoring=MonitoringConfig(**raw.get("monitoring", {})),
            version=raw.get("version", "1.0.0"),
        )

    def validate(self) -> List[str]:
        """Validate configuration and return any errors."""
        errors = []
        if not self.data_sources:
            errors.append("At least one data source is required")
        if self.serving.canary_percentage > 0.5:
            errors.append("Canary percentage should not exceed 50%")
        if self.environment == Environment.PRODUCTION:
            if self.serving.replicas < 2:
                errors.append("Production requires at least 2 replicas")
            if self.monitoring.drift_check_interval_minutes > 60:
                errors.append("Production drift checks must run at least hourly")
        return errors
Architecture Principle

Design for failure from day one. Every network call will eventually time out. Every service will eventually crash. Every model will eventually degrade. Your architecture should handle these cases gracefully through retries, circuit breakers, fallback predictions, and automatic rollback.

2. Data Pipeline Implementation

The data pipeline is the foundation of your platform. It must handle ingestion from heterogeneous sources, validation to catch schema violations and anomalies, transformation to produce features, and versioning to ensure reproducibility. A production pipeline differs from a notebook workflow in one critical way: it must handle failure gracefully and be fully idempotent.

Use a layered approach: raw data lands in a staging area, passes through validation gates, gets transformed into features, and is registered in the feature store. Each step is independently testable and retryable.

from dataclasses import dataclass, field
from typing import Any, Callable, Dict, List, Optional
from datetime import datetime
import hashlib
import json
import logging

logger = logging.getLogger(__name__)


@dataclass
class DataValidationResult:
    """Result of a data validation check."""
    is_valid: bool
    check_name: str
    details: str
    severity: str = "error"  # "error" or "warning"
    timestamp: str = field(default_factory=lambda: datetime.utcnow().isoformat())


class DataValidator:
    """Validates incoming data against schema and quality rules."""

    def __init__(self, schema: Dict[str, Any]):
        self.schema = schema
        self.checks: List[Callable] = []
        self._register_default_checks()

    def _register_default_checks(self):
        """Register built-in validation checks."""
        self.checks.extend([
            self._check_schema_conformance,
            self._check_null_rates,
            self._check_value_ranges,
            self._check_row_count,
        ])

    def _check_schema_conformance(self, df) -> DataValidationResult:
        expected_cols = set(self.schema.get("columns", {}).keys())
        actual_cols = set(df.columns)
        missing = expected_cols - actual_cols
        if missing:
            return DataValidationResult(
                is_valid=False,
                check_name="schema_conformance",
                details=f"Missing columns: {missing}",
            )
        return DataValidationResult(
            is_valid=True,
            check_name="schema_conformance",
            details="All expected columns present",
        )

    def _check_null_rates(self, df) -> DataValidationResult:
        max_null_rate = self.schema.get("max_null_rate", 0.05)
        null_rates = df.isnull().mean()
        violations = null_rates[null_rates > max_null_rate]
        if len(violations) > 0:
            return DataValidationResult(
                is_valid=False,
                check_name="null_rates",
                details=f"High null rates: {violations.to_dict()}",
            )
        return DataValidationResult(
            is_valid=True, check_name="null_rates",
            details=f"All null rates below {max_null_rate}",
        )

    def _check_value_ranges(self, df) -> DataValidationResult:
        range_rules = self.schema.get("value_ranges", {})
        for col, bounds in range_rules.items():
            if col in df.columns:
                min_val, max_val = bounds["min"], bounds["max"]
                out_of_range = ((df[col] < min_val) | (df[col] > max_val)).sum()
                if out_of_range > 0:
                    return DataValidationResult(
                        is_valid=False,
                        check_name="value_ranges",
                        details=f"{col}: {out_of_range} values out of [{min_val}, {max_val}]",
                    )
        return DataValidationResult(
            is_valid=True, check_name="value_ranges",
            details="All values within expected ranges",
        )

    def _check_row_count(self, df) -> DataValidationResult:
        min_rows = self.schema.get("min_rows", 1)
        if len(df) < min_rows:
            return DataValidationResult(
                is_valid=False,
                check_name="row_count",
                details=f"Only {len(df)} rows, minimum is {min_rows}",
            )
        return DataValidationResult(
            is_valid=True, check_name="row_count",
            details=f"{len(df)} rows (minimum: {min_rows})",
        )

    def validate(self, df) -> List[DataValidationResult]:
        """Run all validation checks and return results."""
        results = []
        for check in self.checks:
            try:
                result = check(df)
                results.append(result)
            except Exception as e:
                results.append(DataValidationResult(
                    is_valid=False,
                    check_name=check.__name__,
                    details=f"Check failed with error: {e}",
                ))
        return results


class FeaturePipeline:
    """Orchestrates data ingestion, validation, and feature creation."""

    def __init__(self, config: "PlatformConfig"):
        self.config = config
        self.validator = None
        self.transforms: List[Callable] = []

    def add_transform(self, fn: Callable):
        """Register a feature transformation function."""
        self.transforms.append(fn)
        return self

    def run(self, raw_data) -> Dict[str, Any]:
        """Execute the full pipeline: validate, transform, version."""
        run_id = hashlib.sha256(
            f"{datetime.utcnow().isoformat()}".encode()
        ).hexdigest()[:12]

        logger.info(f"Pipeline run {run_id}: starting validation")

        # Step 1: Validate
        if self.validator:
            results = self.validator.validate(raw_data)
            errors = [r for r in results if not r.is_valid and r.severity == "error"]
            if errors:
                logger.error(f"Validation failed: {[e.details for e in errors]}")
                raise ValueError(f"Data validation failed: {len(errors)} errors")

        # Step 2: Transform
        features = raw_data
        for transform in self.transforms:
            logger.info(f"Pipeline run {run_id}: applying {transform.__name__}")
            features = transform(features)

        # Step 3: Compute fingerprint for versioning
        fingerprint = hashlib.sha256(
            features.to_json().encode()
        ).hexdigest()[:16]

        logger.info(f"Pipeline run {run_id}: complete, fingerprint={fingerprint}")

        return {
            "run_id": run_id,
            "features": features,
            "fingerprint": fingerprint,
            "num_rows": len(features),
            "timestamp": datetime.utcnow().isoformat(),
        }

3. Model Training and Validation

Production model training is not "run training script and hope for the best." It is an automated, reproducible process with experiment tracking, hyperparameter optimization, multi-metric validation, and artifact management. Every training run must be traceable: given a deployed model, you must be able to identify exactly which data, code, and hyperparameters produced it.

The training pipeline should include validation gates that prevent bad models from progressing. A new model must beat the current production model on a held-out test set across multiple metrics before it is eligible for deployment.

Python
from dataclasses import dataclass, field
from typing import Any, Dict, List, Optional, Tuple
from datetime import datetime
import json
import logging

logger = logging.getLogger(__name__)


@dataclass
class ExperimentRecord:
    """Tracks a single training experiment."""
    experiment_id: str
    model_name: str
    hyperparameters: Dict[str, Any]
    metrics: Dict[str, float]
    data_fingerprint: str
    artifacts_path: str
    started_at: str
    completed_at: Optional[str] = None
    status: str = "running"
    tags: Dict[str, str] = field(default_factory=dict)


class ModelTrainer:
    """Manages reproducible model training with experiment tracking."""

    def __init__(self, config: TrainingConfig):
        self.config = config
        self.experiments: List[ExperimentRecord] = []

    def train(self, model, train_data, val_data,
              hyperparameters: Dict[str, Any],
              data_fingerprint: str) -> ExperimentRecord:
        """Train a model with full experiment tracking."""
        exp_id = f"exp_{datetime.utcnow().strftime('%Y%m%d_%H%M%S')}"
        logger.info(f"Starting experiment {exp_id}")

        record = ExperimentRecord(
            experiment_id=exp_id,
            model_name=model.__class__.__name__,
            hyperparameters=hyperparameters,
            metrics={},
            data_fingerprint=data_fingerprint,
            artifacts_path=f"artifacts/{exp_id}",
            started_at=datetime.utcnow().isoformat(),
        )

        try:
            # Apply hyperparameters
            for key, value in hyperparameters.items():
                if hasattr(model, key):
                    setattr(model, key, value)

            # Training loop with early stopping
            best_val_loss = float("inf")
            patience_counter = 0

            for epoch in range(self.config.max_epochs):
                train_loss = model.train_epoch(train_data)
                val_metrics = model.evaluate(val_data)
                val_loss = val_metrics["loss"]

                logger.info(
                    f"Epoch {epoch + 1}: "
                    f"train_loss={train_loss:.4f}, val_loss={val_loss:.4f}"
                )

                if val_loss < best_val_loss:
                    best_val_loss = val_loss
                    patience_counter = 0
                    model.save(record.artifacts_path + "/best_model.pt")
                else:
                    patience_counter += 1
                    if patience_counter >= self.config.early_stopping_patience:
                        logger.info(f"Early stopping at epoch {epoch + 1}")
                        break

            # Final evaluation on validation set
            model.load(record.artifacts_path + "/best_model.pt")
            final_metrics = model.evaluate(val_data)
            record.metrics = final_metrics
            record.status = "completed"

        except Exception as e:
            record.status = "failed"
            record.metrics["error"] = str(e)
            logger.error(f"Experiment {exp_id} failed: {e}")

        record.completed_at = datetime.utcnow().isoformat()
        self.experiments.append(record)
        return record


class ValidationGate:
    """Decides whether a trained model is ready for deployment."""

    def __init__(self, thresholds: Dict[str, float],
                 comparison_model_metrics: Optional[Dict[str, float]] = None):
        self.thresholds = thresholds
        self.comparison_metrics = comparison_model_metrics

    def evaluate(self, experiment: ExperimentRecord) -> Tuple[bool, List[str]]:
        """Check if the model passes all quality gates."""
        reasons = []

        # Check absolute thresholds
        for metric, threshold in self.thresholds.items():
            actual = experiment.metrics.get(metric)
            if actual is None:
                reasons.append(f"Missing metric: {metric}")
            elif metric in ("loss", "error_rate", "latency_ms"):
                if actual > threshold:
                    reasons.append(
                        f"{metric}: {actual:.4f} exceeds threshold {threshold}"
                    )
            else:
                if actual < threshold:
                    reasons.append(
                        f"{metric}: {actual:.4f} below threshold {threshold}"
                    )

        # Check against current production model
        if self.comparison_metrics:
            for metric, prod_value in self.comparison_metrics.items():
                new_value = experiment.metrics.get(metric, 0)
                if metric in ("loss", "error_rate"):
                    if new_value > prod_value * 1.02:
                        reasons.append(
                            f"{metric} regression: {new_value:.4f} vs prod {prod_value:.4f}"
                        )
                else:
                    if new_value < prod_value * 0.98:
                        reasons.append(
                            f"{metric} regression: {new_value:.4f} vs prod {prod_value:.4f}"
                        )

        passed = len(reasons) == 0
        return passed, reasons
Pro Tip

Always log the data fingerprint with every experiment. When a deployed model misbehaves, you need to know whether it was a data issue or a model issue. Without fingerprinting, this question is unanswerable.

4. Deployment and Serving

Deployment is not a single event — it is a controlled rollout with safety nets. Your deployment pipeline should support shadow mode (new model receives traffic but does not serve responses), canary releases (gradual traffic shift), and instant rollback (revert to previous model within seconds). The serving layer must also handle graceful degradation when the model is unavailable.

Python
from dataclasses import dataclass, field
from typing import Any, Dict, List, Optional
from enum import Enum
from datetime import datetime
import logging
import time

logger = logging.getLogger(__name__)


class DeploymentStage(Enum):
    SHADOW = "shadow"
    CANARY = "canary"
    PARTIAL = "partial"
    FULL = "full"
    ROLLED_BACK = "rolled_back"


@dataclass
class DeploymentRecord:
    deployment_id: str
    model_version: str
    stage: DeploymentStage
    traffic_percentage: float
    started_at: str
    health_checks_passed: int = 0
    health_checks_failed: int = 0
    error_rate: float = 0.0


class DeploymentOrchestrator:
    """Manages safe, staged model deployments."""

    def __init__(self, serving_config: ServingConfig):
        self.config = serving_config
        self.current_deployment: Optional[DeploymentRecord] = None
        self.previous_deployment: Optional[DeploymentRecord] = None
        self.rollout_stages = [
            (DeploymentStage.SHADOW, 0.0),
            (DeploymentStage.CANARY, 0.05),
            (DeploymentStage.PARTIAL, 0.25),
            (DeploymentStage.PARTIAL, 0.50),
            (DeploymentStage.FULL, 1.0),
        ]

    def deploy(self, model_version: str,
               experiment: ExperimentRecord) -> DeploymentRecord:
        """Start a new staged deployment."""
        deploy_id = f"deploy_{datetime.utcnow().strftime('%Y%m%d_%H%M%S')}"
        self.previous_deployment = self.current_deployment

        self.current_deployment = DeploymentRecord(
            deployment_id=deploy_id,
            model_version=model_version,
            stage=DeploymentStage.SHADOW,
            traffic_percentage=0.0,
            started_at=datetime.utcnow().isoformat(),
        )

        logger.info(
            f"Deployment {deploy_id}: starting shadow mode "
            f"for model {model_version}"
        )
        return self.current_deployment

    def promote(self) -> DeploymentRecord:
        """Advance to the next rollout stage."""
        if not self.current_deployment:
            raise RuntimeError("No active deployment to promote")

        current_idx = None
        for i, (stage, pct) in enumerate(self.rollout_stages):
            if stage == self.current_deployment.stage \
               and pct == self.current_deployment.traffic_percentage:
                current_idx = i
                break

        if current_idx is None or current_idx >= len(self.rollout_stages) - 1:
            logger.info("Already at full deployment")
            return self.current_deployment

        next_stage, next_pct = self.rollout_stages[current_idx + 1]
        self.current_deployment.stage = next_stage
        self.current_deployment.traffic_percentage = next_pct

        logger.info(
            f"Promoted to {next_stage.value} "
            f"({next_pct * 100:.0f}% traffic)"
        )
        return self.current_deployment

    def rollback(self, reason: str) -> Optional[DeploymentRecord]:
        """Instantly rollback to the previous deployment."""
        if not self.previous_deployment:
            logger.error("No previous deployment to rollback to")
            return None

        logger.warning(
            f"ROLLBACK: {self.current_deployment.deployment_id} "
            f"-> {self.previous_deployment.deployment_id}. "
            f"Reason: {reason}"
        )

        self.current_deployment.stage = DeploymentStage.ROLLED_BACK
        self.current_deployment.traffic_percentage = 0.0
        rolled_back = self.current_deployment
        self.current_deployment = self.previous_deployment
        self.current_deployment.traffic_percentage = 1.0
        self.current_deployment.stage = DeploymentStage.FULL
        return rolled_back

    def health_check(self, error_rate: float, latency_p99_ms: float,
                     serving_config: ServingConfig) -> bool:
        """Run a health check and auto-rollback if thresholds are violated."""
        if not self.current_deployment:
            return True

        self.current_deployment.error_rate = error_rate

        if error_rate > serving_config.rollback_error_threshold:
            self.rollback(
                f"Error rate {error_rate:.3f} exceeds "
                f"threshold {serving_config.rollback_error_threshold}"
            )
            return False

        self.current_deployment.health_checks_passed += 1
        return True

5. Monitoring and Maintenance

A deployed model without monitoring is a ticking time bomb. Your platform must continuously track input data quality, prediction distributions, model performance (when ground truth is available), and system health (latency, throughput, errors). When any metric degrades beyond a threshold, the platform should alert the team and optionally trigger automatic retraining.

Python
from dataclasses import dataclass, field
from typing import Any, Callable, Dict, List, Optional
from collections import deque
from datetime import datetime
import logging
import numpy as np

logger = logging.getLogger(__name__)


class PlatformMonitor:
    """Unified monitoring for the ML platform."""

    def __init__(self, config: MonitoringConfig):
        self.config = config
        self.prediction_buffer = deque(maxlen=10000)
        self.latency_buffer = deque(maxlen=10000)
        self.reference_distribution: Optional[np.ndarray] = None
        self.alert_handlers: List[Callable] = []
        self.metrics_history: List[Dict[str, Any]] = []

    def set_reference(self, reference_data: np.ndarray):
        """Set the reference distribution for drift detection."""
        self.reference_distribution = reference_data
        logger.info(f"Reference distribution set: {len(reference_data)} samples")

    def record_prediction(self, input_features: np.ndarray,
                          prediction: Any, latency_ms: float):
        """Record a prediction for monitoring."""
        self.prediction_buffer.append({
            "features": input_features,
            "prediction": prediction,
            "latency_ms": latency_ms,
            "timestamp": datetime.utcnow().isoformat(),
        })
        self.latency_buffer.append(latency_ms)

    def compute_psi(self, current: np.ndarray,
                    reference: np.ndarray,
                    bins: int = 10) -> float:
        """Population Stability Index for drift detection."""
        eps = 1e-6
        breakpoints = np.percentile(
            reference, np.linspace(0, 100, bins + 1)
        )
        ref_counts = np.histogram(reference, bins=breakpoints)[0]
        cur_counts = np.histogram(current, bins=breakpoints)[0]

        ref_pct = ref_counts / len(reference) + eps
        cur_pct = cur_counts / len(current) + eps

        psi = np.sum((cur_pct - ref_pct) * np.log(cur_pct / ref_pct))
        return float(psi)

    def check_health(self) -> Dict[str, Any]:
        """Run all health checks and return a status report."""
        report = {
            "timestamp": datetime.utcnow().isoformat(),
            "checks": {},
            "alerts": [],
        }

        # Latency check
        if self.latency_buffer:
            latencies = np.array(list(self.latency_buffer))
            p99 = np.percentile(latencies, 99)
            p50 = np.percentile(latencies, 50)
            report["checks"]["latency"] = {
                "p50_ms": round(p50, 2),
                "p99_ms": round(p99, 2),
                "status": "ok" if p99 < self.config.latency_p99_threshold_ms else "warning",
            }
            if p99 >= self.config.latency_p99_threshold_ms:
                report["alerts"].append(
                    f"P99 latency {p99:.1f}ms exceeds threshold "
                    f"{self.config.latency_p99_threshold_ms}ms"
                )

        # Drift check
        if self.reference_distribution is not None and self.prediction_buffer:
            current_features = np.array([
                p["features"] for p in self.prediction_buffer
            ])
            if current_features.ndim == 1:
                psi = self.compute_psi(current_features, self.reference_distribution)
            else:
                psi = np.mean([
                    self.compute_psi(current_features[:, i],
                                    self.reference_distribution[:, i])
                    for i in range(current_features.shape[1])
                ])
            report["checks"]["drift"] = {
                "psi": round(psi, 4),
                "threshold": self.config.psi_threshold,
                "status": "ok" if psi < self.config.psi_threshold else "alert",
            }
            if psi >= self.config.psi_threshold:
                report["alerts"].append(
                    f"Data drift detected: PSI={psi:.4f} "
                    f"(threshold: {self.config.psi_threshold})"
                )

        # Fire alerts
        for alert_msg in report["alerts"]:
            for handler in self.alert_handlers:
                handler(alert_msg)

        self.metrics_history.append(report)
        return report
Common Failure

Do not rely solely on model accuracy as your monitoring metric. Accuracy requires ground truth labels, which often arrive days or weeks after predictions. Monitor proxy metrics (prediction distribution shifts, input feature drift, latency spikes) that are available immediately.

6. Defense Presentation Guide

The capstone culminates in a technical defense where you present your platform to reviewers. This is not a slide deck about theory — it is a live demonstration of a working system, accompanied by documentation that shows you understand both the engineering and the trade-offs. Think of it as a senior engineer explaining their system architecture to a new team member.

Structure your defense around four pillars: Architecture Decisions (why you chose each component), Live Demo (show the system handling real requests), Failure Scenarios (demonstrate graceful degradation and rollback), and Metrics and Monitoring (show your dashboards and alerting).

    architecture_decisions: List[ArchitectureDecision]
    system_diagram_path: str
    api_endpoints: List[Dict[str, str]]
    performance_benchmarks: Dict[str, float]
    failure_scenarios_tested: List[Dict[str, str]]
    fairness_report_path: Optional[str] = None
    model_card_path: Optional[str] = None

    def generate_defense_outline(self) -> str:
        """Generate a structured outline for the defense presentation."""
        sections = []

        # Section 1: Architecture Overview (10 minutes)
        sections.append("## 1. Architecture Overview (10 min)")
        sections.append(f"- System diagram: {self.system_diagram_path}")
        sections.append("- Key architecture decisions:")
        for ad in self.architecture_decisions:
            sections.append(
                f"  - {ad.component}: {ad.decision}"
            )
            sections.append(
                f"    Rationale: {ad.rationale}"
            )

        # Section 2: Live Demo (15 minutes)
        sections.append("\n## 2. Live Demo (15 min)")
        sections.append("- Data ingestion: Show pipeline processing new data")
        sections.append("- Training: Trigger a training run, show experiment tracking")
        sections.append("- Serving: Send requests, show prediction responses")
        sections.append("- API endpoints to demonstrate:")
        for ep in self.api_endpoints:
            sections.append(
                f"  - {ep['method']} {ep['path']}: {ep['description']}"
            )

        # Section 3: Failure Scenarios (10 minutes)
        sections.append("\n## 3. Failure Scenarios (10 min)")
        for scenario in self.failure_scenarios_tested:
            sections.append(
                f"- {scenario['name']}: {scenario['description']}"
            )
            sections.append(
                f"  Expected behavior: {scenario['expected_behavior']}"
            )

        # Section 4: Metrics and Monitoring (10 minutes)
        sections.append("\n## 4. Metrics & Monitoring (10 min)")
        sections.append("- Performance benchmarks:")
        for metric, value in self.performance_benchmarks.items():
            sections.append(f"  - {metric}: {value}")
        sections.append("- Show Grafana dashboards")
        sections.append("- Show alerting rules and on-call runbook")

        # Section 5: Responsible AI (5 minutes)
        sections.append("\n## 5. Responsible AI (5 min)")
        if self.fairness_report_path:
            sections.append(f"- Fairness audit: {self.fairness_report_path}")
        if self.model_card_path:
            sections.append(f"- Model card: {self.model_card_path}")
        sections.append("- Known limitations and ethical considerations")

        # Q&A
        sections.append("\n## 6. Q&A (10 min)")
        sections.append("- Open discussion with reviewers")

        return "\n".join(sections)

    def export_documentation(self, output_path: str):
        """Export the full documentation package as JSON."""
        doc = {
            "project_name": self.project_name,
            "author": self.author,
            "generated_at": datetime.utcnow().isoformat(),
            "architecture_decisions": [
                {
                    "component": ad.component,
                    "decision": ad.decision,
                    "alternatives": ad.alternatives_considered,
                    "rationale": ad.rationale,
                    "trade_offs": ad.trade_offs,
                }
                for ad in self.architecture_decisions
            ],
            "api_endpoints": self.api_endpoints,
            "performance_benchmarks": self.performance_benchmarks,
            "failure_scenarios": self.failure_scenarios_tested,
            "defense_outline": self.generate_defense_outline(),
        }

        with open(output_path, "w") as f:
            json.dump(doc, f, indent=2)

        logger.info(f"Documentation exported to {output_path}")

Defense Success Criteria

The strongest capstone defenses do not just show a working system. They demonstrate what went wrong during development and how problems were solved. Reviewers want to see that you can diagnose issues, make trade-off decisions under constraints, and communicate technical choices clearly.

Assignments

Assignment 1 Level 1 — Foundation

Platform Configuration and Data Pipeline

Build the foundational layers of your ML platform: configuration management and data ingestion.

  • Create a PlatformConfig YAML file and loader for your chosen project domain
  • Implement a DataValidator with at least 5 custom validation checks
  • Build a FeaturePipeline that ingests raw data, validates, transforms, and versions output
  • Write unit tests for every validation check and transformation step
  • Demonstrate idempotency: running the pipeline twice on the same data produces identical results
Done when

Your pipeline processes raw data into versioned features with full validation, and every component has passing tests.

Assignment 2 Level 2 — Professional

Training, Deployment, and Monitoring

Add the model training pipeline, staged deployment system, and monitoring layer to your platform.

  • Implement ModelTrainer with experiment tracking and early stopping
  • Build ValidationGate that compares new models against production baselines
  • Create DeploymentOrchestrator with shadow, canary, partial, and full stages
  • Implement PlatformMonitor with latency tracking and PSI-based drift detection
  • Demonstrate automatic rollback when error rate exceeds the configured threshold
Done when

You can train a model, deploy it through canary stages, and show automatic rollback when injecting errors.

Assignment 3 Level 3 — Production-Ready

Complete Capstone Defense

Integrate all components, prepare documentation, and deliver a technical defense of your platform.

  • Integrate all pipeline stages into a single end-to-end workflow
  • Document at least 3 architecture decisions with rationale and trade-offs
  • Test and document at least 4 failure scenarios with expected recovery behavior
  • Produce a fairness audit report and model card for your deployed model
  • Deliver a 45-minute defense covering architecture, live demo, failure scenarios, and monitoring
Done when

Your defense demonstrates a working end-to-end system with documentation, monitoring, and graceful failure handling. Reviewers can ask any question about your architecture decisions and you can justify each one.

Self-Check

Before submitting your capstone, honestly answer these questions.

Can you trace any deployed prediction back to the exact data, code, and hyperparameters that produced it?

Full traceability requires versioned data (fingerprints), experiment tracking (hyperparameters and metrics), model registry (artifact versioning), and deployment records. Given a prediction, you should be able to identify: which model version served it, which experiment produced that model, which data fingerprint was used for training, and which hyperparameters were set.

Does your system handle failure gracefully, or does it crash and leave users without predictions?

A production system must handle: serving container crashes (load balancer reroutes), model degradation (automatic rollback), data pipeline failures (retry with backoff, alerting), and feature store unavailability (cached features or fallback defaults). No single component failure should cause total system failure.

Could you hand this system to another engineer and have them understand, operate, and extend it?

Operability requires: configuration as code (no manual setup steps), architecture decision records (why, not just what), monitoring dashboards with documented thresholds, runbooks for common failure scenarios, and a model card documenting the model's intended use, limitations, and fairness properties. If another engineer needs to page you at 2 AM to understand the system, the documentation is incomplete.
