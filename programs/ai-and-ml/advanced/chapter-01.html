Chapter 01 · Phase 1: Advanced Architectures

Advanced Neural Architectures

Move beyond vanilla feedforward networks into the architectures that power modern deep learning. Understand why residual connections, attention mechanisms, and principled regularization transformed the field — and learn to design networks that train reliably at scale.

Estimated: 3–4 days
6 Core Concepts

Why This Matters

Deep learning hit a wall in 2014. Networks deeper than 20 layers simply refused to train — gradients vanished, accuracy plateaued, and adding more layers actually made performance worse. Then ResNets shattered that barrier, enabling networks with over 150 layers to train smoothly. The field has never looked back.

Understanding why certain architectures work is the difference between a practitioner who tunes hyperparameters blindly and one who designs systems with intent. This chapter is the foundation for everything that follows. Transformers (Chapter 02), generative models (Chapter 03), and production ML systems (Chapters 05–10) all rely on the architectural patterns introduced here: skip connections, attention, normalization, and multi-task learning.

Key Insight

Architecture design is not about memorizing layer configurations. It is about understanding information flow — how gradients propagate backward, how features compose forward, and where bottlenecks form. Every technique in this chapter solves a specific information flow problem.

Key Concepts

1. ResNets and Skip Connections

The core insight behind Residual Networks (ResNets) is deceptively simple: instead of asking a layer to learn a complete mapping H(x), ask it to learn the residual F(x) = H(x) - x. The output becomes F(x) + x, where x is the identity shortcut that bypasses the layer entirely. This solves the degradation problem because if the optimal mapping is close to the identity, learning a near-zero residual is much easier than learning the full identity mapping through multiple nonlinear layers.

Skip connections solve the vanishing gradient problem because gradients flow directly through the identity path during backpropagation. Even if the learned weights produce near-zero gradients, the shortcut ensures the gradient signal reaches the earliest layers without exponential decay. 

                    ┌─────────────────────────────────┐
                    │        Residual Block            │
                    │                                  │
  Input x ────┬────│──▶ Conv3x3 ──▶ BN ──▶ ReLU      │
              │    │──▶ Conv3x3 ──▶ BN ──▶ (+) ──▶ ReLU ──▶ Output
              │    │                        ▲          │
              └────│────────────────────────┘          │
                   │      (skip / identity)            │
                    └─────────────────────────────────┘

  Output = ReLU(F(x) + x)   where F(x) is the residual mapping
Figure 1: A standard ResNet residual block with identity shortcut connection
python 
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    """Standard residual block with two 3x3 convolutions."""

    def __init__(self, in_channels, out_channels, stride=1):
        super().__init__()

        # Main path: two conv layers with batch norm
        self.conv1 = nn.Conv2d(
            in_channels, out_channels,
            kernel_size=3, stride=stride, padding=1, bias=False
        )
        self.bn1 = nn.BatchNorm2d(out_channels)
        self.relu = nn.ReLU(inplace=True)

        self.conv2 = nn.Conv2d(
            out_channels, out_channels,
            kernel_size=3, stride=1, padding=1, bias=False
        )
        self.bn2 = nn.BatchNorm2d(out_channels)

        # Skip connection: use 1x1 conv if dimensions change
        self.shortcut = nn.Sequential()
        if stride != 1 or in_channels != out_channels:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_channels, out_channels,
                          kernel_size=1, stride=stride, bias=False),
                nn.BatchNorm2d(out_channels)
            )

    def forward(self, x):
        identity = self.shortcut(x)

        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))

        # The magic: add the identity shortcut
        out += identity
        out = self.relu(out)
        return out
Dimension Mismatch

When the skip connection crosses a dimension change (e.g., spatial downsampling or channel expansion), you must use a projection shortcut — typically a 1x1 convolution with matching stride. Without this, the tensor shapes will not align for the addition operation and PyTorch will raise a runtime error.

2. Attention Mechanisms

Attention allows a network to dynamically weight different parts of its input based on relevance. Instead of processing all positions equally, the model learns to focus on the most informative regions. The fundamental operation computes a weighted sum of values, where the weights are derived from the compatibility between queries and keys.

Scaled dot-product attention is the most widely used variant. The scaling factor sqrt(d_k) prevents the dot products from growing large in magnitude, which would push the softmax into regions where it has extremely small gradients. 

  Attention(Q, K, V) = softmax( Q * K^T / sqrt(d_k) ) * V

  Where:
    Q  = query matrix    (what am I looking for?)
    K  = key matrix      (what do I contain?)
    V  = value matrix    (what information do I carry?)
    d_k = dimension of keys (scaling factor prevents softmax saturation)

  Multi-Head: run h parallel attention ops, concatenate, project
  ┌──────┐  ┌──────┐  ┌──────┐       ┌──────┐
  │Head 1│  │Head 2│  │Head 3│  ...  │Head h│
  └──┬───┘  └──┬───┘  └──┬───┘       └──┬───┘
     └─────────┴─────────┴──────┬───────┘
                                ▼
                        Concat + Linear
Figure 2: Scaled dot-product attention and multi-head attention architecture
python 
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class ScaledDotProductAttention(nn.Module):
    """Scaled dot-product attention from 'Attention Is All You Need'."""

    def forward(self, query, key, value, mask=None):
        d_k = query.size(-1)

        # Compute attention scores
        scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)

        # Apply mask (e.g., for padding or causal attention)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))

        # Softmax to get attention weights
        attn_weights = F.softmax(scores, dim=-1)

        # Weighted sum of values
        output = torch.matmul(attn_weights, value)
        return output, attn_weights


class MultiHeadAttention(nn.Module):
    """Multi-head attention: run h parallel attention ops."""

    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0

        self.d_k = d_model // n_heads
        self.n_heads = n_heads

        self.w_q = nn.Linear(d_model, d_model)
        self.w_k = nn.Linear(d_model, d_model)
        self.w_v = nn.Linear(d_model, d_model)
        self.w_o = nn.Linear(d_model, d_model)
        self.attention = ScaledDotProductAttention()

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # Project and reshape for multi-head
        Q = self.w_q(query).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        K = self.w_k(key).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)
        V = self.w_v(value).view(batch_size, -1, self.n_heads, self.d_k).transpose(1, 2)

        # Apply attention
        attn_output, _ = self.attention(Q, K, V, mask)

        # Concatenate heads and project
        attn_output = attn_output.transpose(1, 2).contiguous().view(
            batch_size, -1, self.n_heads * self.d_k
        )
        return self.w_o(attn_output)

3. Architecture Search (NAS)

Neural Architecture Search automates the process of finding optimal network designs. Instead of relying on human intuition to decide how many layers, what kernel sizes, or which connections to use, NAS algorithms explore a search space of possible architectures and select the best performers. The three key components are:

  • Search space — the set of possible architectures (cell-based, hierarchical, or unconstrained)
  • Search strategy — how to explore the space (reinforcement learning, evolutionary algorithms, gradient-based methods like DARTS)
  • Performance estimation — how to evaluate each candidate without full training (weight sharing, early stopping, proxy tasks)
python 
# Simplified DARTS-style differentiable architecture search
import torch
import torch.nn as nn

class MixedOperation(nn.Module):
    """A mixture of candidate operations, weighted by architecture params."""

    def __init__(self, channels):
        super().__init__()
        self.ops = nn.ModuleList([
            nn.Sequential(nn.Conv2d(channels, channels, 3, padding=1),
                          nn.BatchNorm2d(channels), nn.ReLU()),
            nn.Sequential(nn.Conv2d(channels, channels, 5, padding=2),
                          nn.BatchNorm2d(channels), nn.ReLU()),
            nn.Sequential(nn.Conv2d(channels, channels, 1),
                          nn.BatchNorm2d(channels), nn.ReLU()),
            nn.MaxPool2d(3, stride=1, padding=1),
            nn.Identity(),  # skip connection
        ])
        # Architecture parameters (learned via gradient descent)
        self.alphas = nn.Parameter(torch.randn(len(self.ops)))

    def forward(self, x):
        weights = torch.softmax(self.alphas, dim=0)
        return sum(w * op(x) for w, op in zip(weights, self.ops))


class DARTSCell(nn.Module):
    """A cell composed of multiple mixed operations for NAS."""

    def __init__(self, channels, num_nodes=4):
        super().__init__()
        self.num_nodes = num_nodes
        self.edges = nn.ModuleDict()
        for i in range(num_nodes):
            for j in range(i + 2):  # 2 input nodes + previous nodes
                self.edges[f'{j}_{i + 2}'] = MixedOperation(channels)

    def forward(self, s0, s1):
        states = [s0, s1]
        for i in range(self.num_nodes):
            node_input = sum(
                self.edges[f'{j}_{i + 2}'](states[j])
                for j in range(i + 2)
            )
            states.append(node_input)
        return torch.cat(states[2:], dim=1)
Pro Tip

Full NAS is computationally expensive. In practice, start with a proven architecture (ResNet, EfficientNet) and use NAS only when you have exhausted manual tuning. Gradient-based methods like DARTS can search in a single GPU-day, while RL-based NAS may require hundreds of GPU-days.

4. Advanced Regularization

Regularization prevents overfitting by constraining what the model can learn. Modern deep learning uses several complementary techniques that go far beyond basic L2 weight decay. Understanding when to apply each technique — and how they interact — is critical for training deep networks.

python 
import torch
import torch.nn as nn

class DropPath(nn.Module):
    """Stochastic Depth: drop entire residual branches during training.
    Used in Vision Transformers and modern ResNets for implicit ensemble."""

    def __init__(self, drop_prob=0.1):
        super().__init__()
        self.drop_prob = drop_prob

    def forward(self, x):
        if not self.training or self.drop_prob == 0.0:
            return x

        keep_prob = 1 - self.drop_prob
        shape = (x.shape[0],) + (1,) * (x.ndim - 1)
        random_tensor = torch.rand(shape, device=x.device)
        random_tensor = torch.floor(random_tensor + keep_prob)
        return x / keep_prob * random_tensor


class DropBlock2D(nn.Module):
    """DropBlock: structured dropout for convolutional layers.
    Drops contiguous regions instead of individual activations."""

    def __init__(self, drop_prob=0.1, block_size=7):
        super().__init__()
        self.drop_prob = drop_prob
        self.block_size = block_size

    def forward(self, x):
        if not self.training or self.drop_prob == 0.0:
            return x

        # Calculate gamma (probability per element)
        _, _, h, w = x.shape
        gamma = (self.drop_prob / (self.block_size ** 2)) * \
                ((h * w) / ((h - self.block_size + 1) * (w - self.block_size + 1)))

        # Sample mask centers
        mask = (torch.rand_like(x[:, :1, :, :]) < gamma).float()

        # Expand mask centers to blocks using max pooling
        mask = F.max_pool2d(mask, self.block_size, stride=1,
                            padding=self.block_size // 2)
        mask = 1 - mask  # Invert: 1 = keep, 0 = drop

        # Normalize to maintain expected values
        count = mask.numel()
        count_ones = mask.sum()
        return x * mask * count / count_ones


# Normalization comparison
batch_norm  = nn.BatchNorm2d(64)     # Normalizes across batch (CNNs)
layer_norm  = nn.LayerNorm(512)      # Normalizes across features (Transformers)
group_norm  = nn.GroupNorm(32, 64)  # Normalizes within groups (small batch)
instance_norm = nn.InstanceNorm2d(64) # Per-sample, per-channel (style transfer)
When to Use Which Normalization

BatchNorm works best with large batch sizes in CNNs. LayerNorm is the standard for transformers and recurrent networks because it does not depend on batch statistics. GroupNorm is the fallback when batch sizes are too small for BatchNorm to be stable (e.g., object detection with batch size 1–2 per GPU). InstanceNorm excels in style transfer where per-instance statistics carry style information.

5. Multi-Task Learning

Multi-task learning trains a single model on multiple related objectives simultaneously. The shared representations act as an implicit regularizer — the model cannot overfit to one task without degrading performance on others. This is especially powerful when tasks share underlying structure, such as object detection (classification + bounding box regression) or autonomous driving (lane detection + depth estimation + semantic segmentation).

The central challenge is loss balancing. Tasks with higher-magnitude losses naturally dominate training. Manually tuning loss weights is fragile and does not generalize. The solution is to learn the weights automatically using task uncertainty.

python 
import torch
import torch.nn as nn

class MultiTaskModel(nn.Module):
    """Shared backbone with task-specific heads."""

    def __init__(self, backbone_dim=512, num_classes=10, num_attributes=40):
        super().__init__()
        self.backbone = nn.Sequential(
            nn.Linear(784, 1024), nn.ReLU(), nn.Dropout(0.3),
            nn.Linear(1024, backbone_dim), nn.ReLU(), nn.Dropout(0.3),
        )
        self.cls_head = nn.Linear(backbone_dim, num_classes)
        self.attr_head = nn.Linear(backbone_dim, num_attributes)
        self.recon_head = nn.Sequential(
            nn.Linear(backbone_dim, 1024), nn.ReLU(),
            nn.Linear(1024, 784), nn.Sigmoid(),
        )

    def forward(self, x):
        features = self.backbone(x)
        return {
            'class_logits': self.cls_head(features),
            'attributes': torch.sigmoid(self.attr_head(features)),
            'reconstruction': self.recon_head(features),
        }


class UncertaintyLoss(nn.Module):
    """Auto-balancing multi-task loss (Kendall et al., 2018).
    Learns a log-variance per task to weight losses automatically."""

    def __init__(self, num_tasks=3):
        super().__init__()
        self.log_vars = nn.Parameter(torch.zeros(num_tasks))

    def forward(self, losses):
        """losses: list of scalar loss values, one per task."""
        total = 0
        for i, loss in enumerate(losses):
            precision = torch.exp(-self.log_vars[i])
            total += precision * loss + self.log_vars[i]
        return total

6. Architecture Design Principles

After studying hundreds of successful architectures, several design principles emerge consistently. These are not rigid rules but reliable heuristics that experienced practitioners apply when building new networks or adapting existing ones.

  • Start simple, add complexity incrementally — Begin with a minimal architecture. Add layers, skip connections, or attention only when you have evidence they help on your data.
  • Preserve information flow — Use skip connections, normalization, and careful initialization to ensure gradients flow through the entire network.
  • Match architecture to data structure — Convolutions for spatial data, attention for sequences, MLPs for tabular data, and graph neural networks for relational data.
  • Bottleneck for efficiency — Reduce dimensionality before expensive operations (1x1 conv before 3x3 conv in bottleneck blocks).
  • Regularize proportionally to capacity — Deeper and wider networks need more regularization. Scale dropout rates and weight decay with model size.
  • Profile before optimizing — Measure where time is actually spent before changing architecture for speed.
python 
import torch
import torch.nn as nn

class BottleneckBlock(nn.Module):
    """ResNet bottleneck: 1x1 reduce -> 3x3 process -> 1x1 expand.
    3x more parameter-efficient than two 3x3 convolutions."""

    expansion = 4

    def __init__(self, in_ch, bottleneck_ch, stride=1, drop_path_rate=0.1):
        super().__init__()
        out_ch = bottleneck_ch * self.expansion

        self.conv1 = nn.Conv2d(in_ch, bottleneck_ch, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(bottleneck_ch)
        self.conv2 = nn.Conv2d(bottleneck_ch, bottleneck_ch, 3,
                               stride=stride, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(bottleneck_ch)
        self.conv3 = nn.Conv2d(bottleneck_ch, out_ch, 1, bias=False)
        self.bn3 = nn.BatchNorm2d(out_ch)
        self.relu = nn.ReLU(inplace=True)
        self.drop_path = DropPath(drop_path_rate)

        self.shortcut = nn.Sequential()
        if stride != 1 or in_ch != out_ch:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_ch, out_ch, 1, stride=stride, bias=False),
                nn.BatchNorm2d(out_ch),
            )

    def forward(self, x):
        identity = self.shortcut(x)
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.relu(self.bn2(self.conv2(out)))
        out = self.bn3(self.conv3(out))
        out = self.drop_path(out)
        out += identity
        return self.relu(out)
How to choose learning rate schedules for deep architectures

For ResNets and similar architectures, cosine annealing with warm restarts is the most reliable schedule. Start with a linear warmup for the first 5–10% of training steps, then decay following a cosine curve. This avoids the sharp drops of step-based schedules and generally yields better final accuracy.

python 
from torch.optim.lr_scheduler import CosineAnnealingWarmRestarts

optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.01)
scheduler = CosineAnnealingWarmRestarts(optimizer, T_0=10, T_mult=2)

for epoch in range(num_epochs):
    train_one_epoch(model, dataloader, optimizer)
    scheduler.step()
When should you use pre-trained weights vs training from scratch?

Almost always start with pre-trained weights. Transfer learning from ImageNet (for vision) or pre-trained language models (for NLP) gives you better features than random initialization, especially when your dataset is small. Train from scratch only when your domain is radically different from the pre-training data (e.g., satellite imagery, radio spectrograms).

Can you explain why skip connections solve the vanishing gradient problem?

The identity shortcut provides a direct path for gradients during backpropagation. Even when the learned residual function has near-zero gradients, the gradient through the identity path is exactly 1. This means the gradient signal reaches the earliest layers without exponential decay.

Can you implement multi-head attention from scratch without reference code?

You should be able to write the query/key/value projections, the scaled dot-product computation, the softmax normalization, and the head concatenation from memory. If not, re-implement it from scratch until it becomes natural.

Do you understand when to use BatchNorm vs LayerNorm vs GroupNorm?

BatchNorm for large-batch CNNs, LayerNorm for transformers and RNNs, GroupNorm for small-batch scenarios. The key insight is what dimension each normalizes over. If you cannot draw the normalization axis for each variant, review the concept.

Can you design a multi-task loss function that balances tasks automatically?

Uncertainty weighting (learning a log-variance per task) is the standard approach. You should understand why fixed weights fail and how learned precision parameters solve this by downweighting noisier tasks automatically.
