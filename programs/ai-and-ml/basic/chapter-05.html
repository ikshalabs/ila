Neural Network Foundations

Understand how neural networks learn from data — from the mathematics of a single neuron through backpropagation to building multi-layer networks with PyTorch. This chapter bridges the gap between classical machine learning and the deep learning revolution.

Estimated: 4–5 days
6 Key Concepts
3 Assignments

Why This Matters

Neural networks are the engine behind virtually every breakthrough in modern AI — from image recognition that outperforms radiologists, to language models that write coherent prose, to systems that fold proteins and discover new drugs. Understanding how these networks actually learn is not optional for any serious practitioner. Without it, you are simply pressing buttons in a framework and hoping for the best.

The concepts in this chapter — forward propagation, loss functions, gradient descent, and backpropagation — form the computational backbone of every deep learning system ever built. Whether you end up working with convolutional networks for vision, transformers for language, or reinforcement learning for control, the same fundamental learning algorithm drives them all.

More practically, understanding these foundations lets you debug models that fail. When your loss curve plateaus, when gradients vanish, when your model memorizes training data instead of generalizing — knowing the mechanics beneath the surface is the difference between being stuck and knowing exactly which lever to pull.

Key Insight

A neural network is just a differentiable function composed of simple operations (matrix multiplications and element-wise nonlinearities). Training is the process of computing how much each parameter contributed to the error, then nudging it in the opposite direction. Every framework — PyTorch, TensorFlow, JAX — automates this with automatic differentiation, but understanding the chain rule that powers it is what separates engineers from practitioners.

Key Concepts

1. The Single Neuron: Perceptron and Activation Functions

Every neural network begins with the humble neuron — a computational unit that takes a weighted sum of its inputs, adds a bias term, and passes the result through a nonlinear activation function. Mathematically, a single neuron computes: y = f(w · x + b), where w is a weight vector, x is the input vector, b is a scalar bias, and f is the activation function.

The original perceptron used a step function as its activation: output 1 if the weighted sum exceeds a threshold, 0 otherwise. While historically important, step functions are not differentiable and therefore cannot be trained with gradient descent. Modern networks use smooth, differentiable activations that allow gradients to flow through the computation graph.

python 
import numpy as np
import matplotlib.pyplot as plt

# Common activation functions and their derivatives

def sigmoid(z):
    """Maps any real number to (0, 1). Used for binary classification output."""
    return 1 / (1 + np.exp(-z))

def sigmoid_derivative(z):
    s = sigmoid(z)
    return s * (1 - s)

def relu(z):
    """Rectified Linear Unit: max(0, z). The workhorse of modern networks."""
    return np.maximum(0, z)

def relu_derivative(z):
    return (z > 0).astype(np.float64)

def tanh_activation(z):
    """Maps to (-1, 1). Zero-centered, often better than sigmoid for hidden layers."""
    return np.tanh(z)

def leaky_relu(z, alpha=0.01):
    """Allows small negative gradients to prevent dead neurons."""
    return np.where(z > 0, z, alpha * z)

# A single neuron forward pass
def neuron(x, w, b, activation=relu):
    """Single neuron: weighted sum + bias + activation."""
    z = np.dot(w, x) + b          # Linear combination
    a = activation(z)              # Nonlinear activation
    return a, z

# Example: 3-input neuron
x = np.array([0.5, -0.3, 0.8])     # Input features
w = np.array([0.4, 0.7, -0.2])    # Learned weights
b = 0.1                             # Learned bias

output, pre_activation = neuron(x, w, b)
print(f"Pre-activation (z): {pre_activation:.4f}")
print(f"Output (a):         {output:.4f}")
Why Nonlinearity Is Essential

Without activation functions, a neural network of any depth is just a single linear transformation: W3 * (W2 * (W1 * x)) = W_combined * x. The activation function is what gives neural networks their power to approximate any continuous function (the Universal Approximation Theorem). Remove it, and you have a very expensive linear regression.

2. Forward Propagation: Data Flows Through the Network

A neural network is built by stacking layers of neurons. In forward propagation, data enters the input layer, passes through one or more hidden layers, and exits through the output layer. At each layer, the computation is: apply a linear transformation (matrix multiply plus bias), then apply an activation function. The output of one layer becomes the input to the next. 

  INPUT LAYER        HIDDEN LAYER 1      HIDDEN LAYER 2      OUTPUT LAYER
  (3 neurons)        (4 neurons)         (4 neurons)         (1 neuron)

    x1 ----\         h1_1 ----\          h2_1 ----\
             \---->  h1_2 ----  \---->   h2_2 ----  \---->   y (prediction)
    x2 ----/  \-->   h1_3 ----/  \-->    h2_3 ----/  \-->
             /       h1_4 ----    /      h2_4 ----    /
    x3 ----/                    /                    /

  z1 = W1*x + b1    z2 = W2*h1 + b2    z3 = W3*h2 + b3
  h1 = ReLU(z1)     h2 = ReLU(z2)      y  = sigmoid(z3)
Fig 1. A 3-layer feedforward neural network. Each arrow represents a learned weight parameter.
python 
import numpy as np

class NeuralNetwork:
    """A simple feedforward neural network from scratch."""

    def __init__(self, layer_sizes):
        """
        layer_sizes: list of integers, e.g. [3, 4, 4, 1]
        defines input_dim, hidden layers, and output_dim.
        """
        self.weights = []
        self.biases = []
        self.num_layers = len(layer_sizes)

        # Xavier/Glorot initialization for better training
        for i in range(len(layer_sizes) - 1):
            fan_in = layer_sizes[i]
            fan_out = layer_sizes[i + 1]
            scale = np.sqrt(2.0 / (fan_in + fan_out))
            W = np.random.randn(fan_out, fan_in) * scale
            b = np.zeros((fan_out, 1))
            self.weights.append(W)
            self.biases.append(b)

    def forward(self, X):
        """Forward pass: returns prediction and caches values for backprop."""
        self.cache = {'A0': X}
        A = X

        # Hidden layers use ReLU activation
        for i in range(len(self.weights) - 1):
            Z = self.weights[i] @ A + self.biases[i]
            A = np.maximum(0, Z)  # ReLU
            self.cache[f'Z{i+1}'] = Z
            self.cache[f'A{i+1}'] = A

        # Output layer uses sigmoid for binary classification
        Z_out = self.weights[-1] @ A + self.biases[-1]
        A_out = 1 / (1 + np.exp(-Z_out))  # Sigmoid
        self.cache[f'Z{len(self.weights)}'] = Z_out
        self.cache[f'A{len(self.weights)}'] = A_out

        return A_out

# Create a network: 3 inputs -> 4 hidden -> 4 hidden -> 1 output
net = NeuralNetwork([3, 4, 4, 1])

# Forward pass with 5 samples
X = np.random.randn(3, 5)  # 3 features, 5 samples (column-major)
predictions = net.forward(X)
print(f"Predictions shape: {predictions.shape}")  # (1, 5)
print(f"Predictions: {predictions.round(3)}")

3. Loss Functions: Measuring How Wrong We Are

The loss function (also called the cost function or objective function) quantifies the difference between the network's predictions and the true labels. Training a neural network means minimizing this function. The choice of loss function depends on the task: binary cross-entropy for binary classification, categorical cross-entropy for multi-class, and mean squared error for regression.

The loss must be differentiable with respect to the network's parameters, because we need to compute gradients to update those parameters. A good loss function is also numerically stable — it should not produce NaN or infinity values during training, even for extreme predictions.

python 
import numpy as np

# Binary Cross-Entropy Loss (for binary classification)
def binary_cross_entropy(y_true, y_pred, epsilon=1e-15):
    """
    y_true: ground truth labels (0 or 1), shape (1, m)
    y_pred: predicted probabilities, shape (1, m)
    epsilon: small value to prevent log(0)
    """
    y_pred = np.clip(y_pred, epsilon, 1 - epsilon)
    m = y_true.shape[1]
    loss = -(1 / m) * np.sum(
        y_true * np.log(y_pred) + (1 - y_true) * np.log(1 - y_pred)
    )
    return loss

# Mean Squared Error Loss (for regression)
def mse_loss(y_true, y_pred):
    """Average squared difference between prediction and truth."""
    m = y_true.shape[1]
    loss = (1 / (2 * m)) * np.sum((y_pred - y_true) ** 2)
    return loss

# Categorical Cross-Entropy (for multi-class classification)
def categorical_cross_entropy(y_true_onehot, y_pred, epsilon=1e-15):
    """
    y_true_onehot: one-hot encoded labels, shape (num_classes, m)
    y_pred: softmax probabilities, shape (num_classes, m)
    """
    y_pred = np.clip(y_pred, epsilon, 1 - epsilon)
    m = y_true_onehot.shape[1]
    loss = -(1 / m) * np.sum(y_true_onehot * np.log(y_pred))
    return loss

# Example: comparing predictions to true labels
y_true = np.array([[1, 0, 1, 1, 0]])
y_pred = np.array([[0.9, 0.1, 0.8, 0.7, 0.3]])

loss = binary_cross_entropy(y_true, y_pred)
print(f"Binary Cross-Entropy Loss: {loss:.4f}")  # ~0.2197
Pro Tip

The epsilon clipping in cross-entropy is critical for numerical stability. Without it, np.log(0) produces -inf, which poisons the entire gradient computation. In production frameworks like PyTorch, nn.BCEWithLogitsLoss combines sigmoid and cross-entropy in a single numerically stable operation — always prefer it over computing sigmoid and cross-entropy separately.

4. Backpropagation: The Chain Rule at Scale

Backpropagation is the algorithm that computes the gradient of the loss with respect to every parameter in the network. It works by applying the chain rule of calculus layer by layer, starting from the output and propagating backward. This is not just a theoretical concept — it is exactly what happens inside loss.backward() in PyTorch.

The key insight is that the gradient at each layer depends only on the gradient from the layer above it and the local values cached during the forward pass. This means we can compute all gradients in a single backward pass through the network, making it computationally efficient — roughly the same cost as the forward pass. 

  FORWARD PASS (left to right):
  x ---> [z = Wx + b] ---> [a = ReLU(z)] ---> [z = Wa + b] ---> [L = loss(y, a)]

  BACKWARD PASS (right to left):
  dL/dx <--- [dL/dW, dL/db] <--- [dL/dz * ReLU'(z)] <--- [dL/dW, dL/db] <--- dL/dL = 1

  At each layer:
    dL/dW = dL/dz * a_prev^T       (gradient w.r.t. weights)
    dL/db = sum(dL/dz, axis=1)      (gradient w.r.t. biases)
    dL/da_prev = W^T * dL/dz        (gradient to pass backward)
Fig 2. Backpropagation computes gradients by reversing the computation graph using the chain rule.
python 
def backward(net, y_true):
    """Compute gradients via backpropagation."""
    m = y_true.shape[1]
    L = len(net.weights)
    grads = {}

    # Gradient of loss w.r.t. output (binary cross-entropy + sigmoid)
    A_out = net.cache[f'A{L}']
    dZ = A_out - y_true  # Simplified gradient for sigmoid + BCE

    # Backpropagate through each layer
    for i in reversed(range(L)):
        A_prev = net.cache[f'A{i}']

        # Gradients for this layer's parameters
        grads[f'dW{i+1}'] = (1 / m) * (dZ @ A_prev.T)
        grads[f'db{i+1}'] = (1 / m) * np.sum(dZ, axis=1, keepdims=True)

        # Gradient to pass to previous layer
        if i > 0:
            dA_prev = net.weights[i].T @ dZ
            Z_prev = net.cache[f'Z{i}']
            dZ = dA_prev * (Z_prev > 0)  # ReLU derivative

    return grads

def update_parameters(net, grads, learning_rate=0.01):
    """Gradient descent: nudge each parameter opposite to its gradient."""
    for i in range(len(net.weights)):
        net.weights[i] -= learning_rate * grads[f'dW{i+1}']
        net.biases[i] -= learning_rate * grads[f'db{i+1}']

# Full training loop from scratch
net = NeuralNetwork([2, 8, 4, 1])
losses = []

for epoch in range(1000):
    predictions = net.forward(X_train)
    loss = binary_cross_entropy(y_train, predictions)
    grads = backward(net, y_train)
    update_parameters(net, grads, learning_rate=0.01)
    losses.append(loss)

    if epoch % 100 == 0:
        print(f"Epoch {epoch}, Loss: {loss:.4f}")
Vanishing and Exploding Gradients

In deep networks, gradients can shrink exponentially (vanish) or grow exponentially (explode) as they propagate backward through many layers. Vanishing gradients mean early layers learn extremely slowly. Exploding gradients cause numerical overflow and unstable training. Solutions include ReLU activations (which do not saturate for positive values), careful weight initialization (Xavier/He), batch normalization, and gradient clipping.

5. Gradient Descent Optimizers: Beyond Vanilla SGD

Vanilla gradient descent updates every parameter by subtracting the gradient scaled by a learning rate. While correct, it has several shortcomings: it can oscillate in ravines, it uses the same learning rate for all parameters, and it can get stuck in saddle points. Modern optimizers address these issues with momentum and adaptive learning rates.

python 
import numpy as np

# SGD with Momentum — smooths out oscillations
class MomentumSGD:
    def __init__(self, params, lr=0.01, momentum=0.9):
        self.lr = lr
        self.momentum = momentum
        self.velocity = [np.zeros_like(p) for p in params]

    def step(self, params, grads):
        for i, (p, g) in enumerate(zip(params, grads)):
            self.velocity[i] = self.momentum * self.velocity[i] - self.lr * g
            p += self.velocity[i]

# Adam — the default optimizer for most deep learning tasks
class Adam:
    def __init__(self, params, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8):
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.eps = eps
        self.m = [np.zeros_like(p) for p in params]  # First moment (mean)
        self.v = [np.zeros_like(p) for p in params]  # Second moment (variance)
        self.t = 0

    def step(self, params, grads):
        self.t += 1
        for i, (p, g) in enumerate(zip(params, grads)):
            # Update biased first and second moment estimates
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * g
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * g**2

            # Bias correction (critical in early steps)
            m_hat = self.m[i] / (1 - self.beta1**self.t)
            v_hat = self.v[i] / (1 - self.beta2**self.t)

            # Parameter update with adaptive learning rate
            p -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)

Adam (Adaptive Moment Estimation) combines the benefits of momentum (first moment) with adaptive per-parameter learning rates (second moment). It is the default optimizer for most deep learning tasks because it works well out of the box with minimal hyperparameter tuning. The typical starting learning rate for Adam is 0.001, compared to 0.01 or 0.1 for vanilla SGD.

6. Building Neural Networks with PyTorch

While understanding the math from scratch is essential, production neural networks are built with frameworks that provide automatic differentiation, GPU acceleration, and pre-built layers. PyTorch is the industry standard for research and increasingly for production. Its autograd engine computes all the gradients we painstakingly derived above — automatically.

python 
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset

# Define a neural network using nn.Module
class BinaryClassifier(nn.Module):
    def __init__(self, input_dim, hidden_dim=64):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.BatchNorm1d(hidden_dim),
            nn.Dropout(0.3),

            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.BatchNorm1d(hidden_dim),
            nn.Dropout(0.3),

            nn.Linear(hidden_dim, 1),
        )

    def forward(self, x):
        return self.network(x)

# Create model, loss, and optimizer
model = BinaryClassifier(input_dim=10)
criterion = nn.BCEWithLogitsLoss()  # Numerically stable sigmoid + BCE
optimizer = optim.Adam(model.parameters(), lr=0.001)

# Prepare data
X_train = torch.randn(1000, 10)
y_train = torch.randint(0, 2, (1000, 1)).float()
dataset = TensorDataset(X_train, y_train)
loader = DataLoader(dataset, batch_size=32, shuffle=True)

# Training loop — the standard PyTorch pattern
for epoch in range(50):
    epoch_loss = 0.0
    model.train()  # Enable dropout and batch norm training mode

    for batch_X, batch_y in loader:
        # Forward pass
        logits = model(batch_X)
        loss = criterion(logits, batch_y)

        # Backward pass — the three essential lines
        optimizer.zero_grad()   # Clear previous gradients
        loss.backward()         # Compute gradients via autograd
        optimizer.step()        # Update parameters

        epoch_loss += loss.item()

    if (epoch + 1) % 10 == 0:
        avg_loss = epoch_loss / len(loader)
        print(f"Epoch {epoch+1}/50, Loss: {avg_loss:.4f}")
The Three-Line Training Pattern

Every PyTorch training loop follows the same three essential lines: optimizer.zero_grad() to clear accumulated gradients, loss.backward() to compute new gradients via backpropagation, and optimizer.step() to update the parameters. Forgetting zero_grad() is the most common PyTorch bug — gradients accumulate across batches, causing erratic training behavior.

python 
# Evaluating the trained model
model.eval()  # Switch to eval mode (disables dropout, uses running BN stats)

with torch.no_grad():  # No gradient computation needed for inference
    X_test = torch.randn(200, 10)
    logits = model(X_test)
    probabilities = torch.sigmoid(logits)
    predictions = (probabilities > 0.5).float()
    print(f"Predicted {predictions.sum().item():.0f} positives out of {len(predictions)}")

# Inspecting model architecture and parameter count
total_params = sum(p.numel() for p in model.parameters())
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f"Total parameters: {total_params:,}")
print(f"Trainable: {trainable:,}")

# Saving and loading models
torch.save(model.state_dict(), "model_weights.pth")

# Load later:
loaded = BinaryClassifier(input_dim=10)
loaded.load_state_dict(torch.load("model_weights.pth"))
loaded.eval()
Pro Tip: Regularization Techniques

The model above includes two critical regularization techniques: Dropout randomly zeroes out neurons during training (forcing redundancy), and Batch Normalization normalizes layer inputs to have zero mean and unit variance (stabilizing training). Together, they significantly reduce overfitting. Always call model.eval() before inference to disable dropout and use population statistics for batch norm.

Self-Check

Before moving to Chapter 06, honestly answer these questions. If any answer is "no," revisit the relevant section.

Can you explain what happens during a single training step (forward, loss, backward, update)?

A training step has four phases: (1) Forward pass — input data flows through the network layer by layer, producing a prediction. (2) Loss computation — the prediction is compared to the true label using a differentiable loss function. (3) Backward pass — gradients of the loss w.r.t. every parameter are computed via the chain rule (backpropagation). (4) Parameter update — each parameter is nudged in the direction that reduces the loss, scaled by the learning rate. If you can draw this on a whiteboard, you are ready.

Do you understand why ReLU largely replaced sigmoid in hidden layers?

Sigmoid squashes all inputs to (0, 1), so for very large or very small inputs, the gradient is nearly zero — this is the vanishing gradient problem. ReLU has a constant gradient of 1 for positive inputs, allowing gradients to flow freely through deep networks. ReLU is also computationally cheaper (a simple threshold vs. an exponential). The tradeoff is "dead neurons" (permanently zero output), which Leaky ReLU addresses.

Can you write a PyTorch training loop from memory?

The core loop should include: iterating over DataLoader batches, calling model(batch_x) for the forward pass, computing loss with the criterion, calling optimizer.zero_grad(), loss.backward(), and optimizer.step(). You should also know to call model.train() before training and model.eval() before evaluation, and to wrap inference in torch.no_grad().

Can you describe what Adam optimizer does differently from vanilla SGD?

Adam maintains two running averages: the first moment (mean of gradients, like momentum) and the second moment (mean of squared gradients, for adaptive learning rates). It also applies bias correction since both estimates start at zero. This means Adam adapts the learning rate per parameter — parameters with consistently large gradients get smaller updates, and vice versa. The result is faster convergence with less hyperparameter sensitivity.

Do you understand the purpose of BatchNorm and Dropout, and when each is active?

Batch Normalization normalizes each layer's input to zero mean and unit variance using batch statistics during training, and learned population statistics during inference. It stabilizes and accelerates training. Dropout randomly sets a fraction of neuron outputs to zero during training, forcing the network to learn redundant representations. During inference, dropout is disabled and outputs are scaled. Both behaviors change between model.train() and model.eval() modes.
