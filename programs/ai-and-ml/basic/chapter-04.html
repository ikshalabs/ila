Chapter 04 · Phase 2: Classical ML

Ensemble Methods and Model Selection

Combine multiple weak learners into powerful ensembles with random forests and gradient boosting, then systematically find the best model using cross-validation and hyperparameter tuning.

Estimated: 3–4 days
6 Core Concepts
3 Assignments

Why This Matters

Single models have inherent limitations — a decision tree overfits, a linear model underfits complex patterns. Ensemble methods solve this by combining multiple models, each capturing different aspects of the data. The result is almost always superior to any individual model. In practice, gradient boosting frameworks like XGBoost and LightGBM dominate machine learning competitions and production systems alike.

But building powerful models is only half the battle. You also need systematic methods to select the best model and tune its parameters without overfitting to your validation set. Cross-validation and hyperparameter search are the tools that bridge the gap between experimentation and production-ready ML.

Key Insight

The "wisdom of crowds" principle applies to ML: a committee of diverse, independently-trained models will outperform any single expert model. The key word is diverse — if all models make the same errors, ensembling provides no benefit.

Key Concepts

1. Bagging and Random Forests

Bagging (Bootstrap Aggregating) trains multiple models on random subsets of the training data, then averages their predictions (regression) or takes a majority vote (classification). This reduces variance without increasing bias. Random Forests extend bagging by adding feature randomness: at each split, only a random subset of features is considered. This decorrelates the trees, making the ensemble more powerful. 

  Training Data
       |
       +-- Bootstrap Sample 1 --> Tree 1 --+
       +-- Bootstrap Sample 2 --> Tree 2 --+
       +-- Bootstrap Sample 3 --> Tree 3 --+--> Aggregate --> Final Prediction
       +-- Bootstrap Sample 4 --> Tree 4 --+    (vote or
       +-- Bootstrap Sample N --> Tree N --+     average)

  Each tree sees ~63.2% of samples (bootstrap)
  Each split considers sqrt(n_features) features (random forest)
Figure 1: Random Forest builds multiple trees on bootstrapped data and aggregates predictions
python 
from sklearn.ensemble import RandomForestClassifier
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split, cross_val_score
import numpy as np

# Load data
data = load_breast_cancer()
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.2, random_state=42, stratify=data.target
)

# Train a Random Forest
rf = RandomForestClassifier(
    n_estimators=200,           # Number of trees
    max_depth=10,               # Limit depth per tree
    max_features='sqrt',        # Features per split
    min_samples_leaf=5,         # Min samples in leaf
    oob_score=True,             # Out-of-bag estimate
    n_jobs=-1,                  # Use all CPU cores
    random_state=42
)

# Cross-validation for robust estimate
cv_scores = cross_val_score(rf, X_train, y_train, cv=5, scoring='accuracy')
print(f"CV Accuracy: {cv_scores.mean():.4f} (+/- {cv_scores.std():.4f})")

# Fit and evaluate
rf.fit(X_train, y_train)
print(f"OOB Score:   {rf.oob_score_:.4f}")
print(f"Test Acc:    {rf.score(X_test, y_test):.4f}")

2. Gradient Boosting (XGBoost, LightGBM)

While bagging trains trees independently and averages them, boosting trains trees sequentially, where each new tree corrects the errors of the previous ones. Gradient boosting specifically fits each new tree to the residual errors (negative gradient of the loss function) of the current ensemble.

python 
import xgboost as xgb
import lightgbm as lgb

# --- XGBoost with early stopping ---
xgb_model = xgb.XGBClassifier(
    n_estimators=500,
    learning_rate=0.05,
    max_depth=4,
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=0.1,
    reg_lambda=1.0,
    eval_metric='logloss',
    early_stopping_rounds=20,
    random_state=42
)

xgb_model.fit(
    X_train, y_train,
    eval_set=[(X_test, y_test)],
    verbose=50
)
print(f"XGBoost best iteration: {xgb_model.best_iteration}")
print(f"XGBoost test accuracy:  {xgb_model.score(X_test, y_test):.4f}")

# --- LightGBM ---
lgb_model = lgb.LGBMClassifier(
    n_estimators=500,
    learning_rate=0.05,
    num_leaves=31,
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=0.1,
    reg_lambda=1.0,
    random_state=42,
    verbose=-1
)

lgb_model.fit(
    X_train, y_train,
    eval_set=[(X_test, y_test)],
    callbacks=[lgb.early_stopping(20), lgb.log_evaluation(50)]
)
print(f"LightGBM test accuracy: {lgb_model.score(X_test, y_test):.4f}")
Learning Rate and Number of Trees

There is a direct tradeoff between learning_rate and n_estimators. A smaller learning rate requires more trees but typically produces better results. Use early stopping to find the optimal number of trees automatically instead of tuning it manually.

3. Cross-Validation Strategies

A single train-test split can give misleading results. K-fold cross-validation rotates through K different splits, training on K-1 folds and validating on the remaining fold. For classification, always use StratifiedKFold to preserve class ratios.

python 
from sklearn.model_selection import (
    StratifiedKFold, RepeatedStratifiedKFold, cross_validate
)

# Stratified K-Fold preserves class ratios
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

# Repeated CV for more robust estimates
rskf = RepeatedStratifiedKFold(n_splits=5, n_repeats=3, random_state=42)

# cross_validate returns multiple metrics
results = cross_validate(
    rf, X_train, y_train,
    cv=skf,
    scoring=['accuracy', 'f1', 'roc_auc'],
    return_train_score=True
)

print("Metric          | Train            | Validation")
print("-" * 55)
for metric in ['accuracy', 'f1', 'roc_auc']:
    tr = results[f'train_{metric}']
    va = results[f'test_{metric}']
    print(f"{metric:16s}| {tr.mean():.4f} +/- {tr.std():.3f} | {va.mean():.4f} +/- {va.std():.3f}")

4. Hyperparameter Tuning (GridSearch, RandomSearch, Bayesian)

Finding the right hyperparameter combination is critical. GridSearchCV tries all combinations exhaustively. RandomizedSearchCV samples randomly and is faster for large spaces. Optuna uses Bayesian optimization to focus on promising regions.

python 
from sklearn.model_selection import GridSearchCV, RandomizedSearchCV
from scipy.stats import randint, uniform

# GridSearchCV: Exhaustive but slow
param_grid = {
    'n_estimators': [100, 200, 300],
    'max_depth': [3, 5, 7, 10],
    'min_samples_leaf': [1, 2, 5],
}

grid_search = GridSearchCV(
    RandomForestClassifier(random_state=42),
    param_grid, cv=5, scoring='f1', n_jobs=-1
)
grid_search.fit(X_train, y_train)
print(f"Grid Best F1:  {grid_search.best_score_:.4f}")
print(f"Grid Params:   {grid_search.best_params_}")
python 
# Bayesian Optimization with Optuna
import optuna

def objective(trial):
    params = {
        'n_estimators': trial.suggest_int('n_estimators', 50, 500),
        'max_depth': trial.suggest_int('max_depth', 2, 15),
        'learning_rate': trial.suggest_float('learning_rate', 0.01, 0.3, log=True),
        'subsample': trial.suggest_float('subsample', 0.5, 1.0),
        'colsample_bytree': trial.suggest_float('colsample_bytree', 0.5, 1.0),
    }

    model = xgb.XGBClassifier(**params, random_state=42, eval_metric='logloss')
    scores = cross_val_score(model, X_train, y_train, cv=5, scoring='f1')
    return scores.mean()

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=50, show_progress_bar=True)
print(f"Best F1:     {study.best_value:.4f}")
print(f"Best params: {study.best_params}")

5. Model Comparison and Selection

With multiple tuned models, run the same cross-validation procedure on each, collect the scores, and compare. Always report confidence intervals, not just point estimates. A box plot makes differences between models immediately visible.

python 
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.linear_model import LogisticRegression
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

models = {
    'LR': Pipeline([('s', StandardScaler()), ('m', LogisticRegression(max_iter=1000))]),
    'SVM': Pipeline([('s', StandardScaler()), ('m', SVC())]),
    'RF': RandomForestClassifier(n_estimators=200, random_state=42),
    'XGB': xgb.XGBClassifier(n_estimators=200, eval_metric='logloss', random_state=42),
    'LGBM': lgb.LGBMClassifier(n_estimators=200, random_state=42, verbose=-1),
}

results = {}
for name, model in models.items():
    scores = cross_val_score(model, X_train, y_train, cv=5, scoring='f1')
    results[name] = scores
    print(f"{name:6s} | F1: {scores.mean():.4f} +/- {scores.std():.4f}")

# Box plot comparison
fig, ax = plt.subplots(figsize=(10, 5))
ax.boxplot(results.values(), labels=results.keys())
ax.set_ylabel('F1 Score')
ax.set_title('Model Comparison: 5-Fold CV')
plt.tight_layout()
plt.show()

6. Feature Importance and Interpretability

Understanding why a model makes predictions is critical. Tree-based models provide built-in feature importance. For model-agnostic interpretability, permutation importance measures how much performance drops when a feature's values are shuffled.

python 
from sklearn.inspection import permutation_importance
import pandas as pd

rf.fit(X_train, y_train)

# Built-in Gini importance
gini_imp = pd.Series(rf.feature_importances_, index=data.feature_names)
gini_imp = gini_imp.sort_values(ascending=False)

# Permutation importance (more reliable)
perm_result = permutation_importance(rf, X_test, y_test, n_repeats=10, random_state=42)
perm_imp = pd.Series(perm_result.importances_mean, index=data.feature_names)
perm_imp = perm_imp.sort_values(ascending=False)

# Side-by-side comparison
fig, axes = plt.subplots(1, 2, figsize=(16, 6))
gini_imp[:15].plot(kind='barh', ax=axes[0], color='teal')
axes[0].set_title('Gini Importance (Top 15)')
perm_imp[:15].plot(kind='barh', ax=axes[1], color='coral')
axes[1].set_title('Permutation Importance (Top 15)')
plt.tight_layout()
plt.show()
Why does Gini importance disagree with permutation importance?

Gini importance is biased toward high-cardinality features that offer more split opportunities. Permutation importance measures actual predictive contribution on the test set. When they disagree, trust permutation importance for feature selection decisions.

How do I choose between XGBoost and LightGBM?

LightGBM is typically faster on large datasets due to histogram-based binning. XGBoost has a longer track record and more documentation. Performance is often similar. Try both and let cross-validation decide.

What about early stopping for gradient boosting?

Early stopping monitors validation loss and stops when it stops improving. This prevents overfitting and automatically selects the optimal number of trees. Far more efficient than tuning n_estimators manually.

Can you explain the difference between bagging and boosting?

Bagging trains models independently on bootstrapped data and averages them (reduces variance). Boosting trains models sequentially, each correcting the previous one's errors (reduces bias). Random Forest = bagging. XGBoost/LightGBM = gradient boosting.

Do you know when to use GridSearch vs RandomSearch vs Bayesian optimization?

GridSearch: small spaces, exhaustive. RandomSearch: larger spaces, limited budget. Bayesian (Optuna): large spaces with expensive evaluations, intelligently focuses on promising regions.

Can you interpret feature importance and explain model predictions?

You should extract both Gini and permutation importance, understand their differences, and communicate which features matter most. Trust permutation importance when they disagree.
