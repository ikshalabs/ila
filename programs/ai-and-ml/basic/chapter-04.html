Chapter 04 · Phase 2: Core Machine Learning

Unsupervised Learning

Discover hidden structure in unlabeled data through clustering, dimensionality reduction, and anomaly detection — essential techniques for exploratory analysis and data preprocessing.

Estimated: 3–4 days
3 Advancement Levels
3 Assignments

Why This Matters

Not all data comes with labels. In fact, most real-world data is unlabeled. Unsupervised learning discovers hidden structure in data without guidance — finding customer segments, reducing dimensionality for visualization, detecting anomalies, and compressing information. These techniques are essential for exploratory analysis and often serve as preprocessing steps for supervised models.

Companies use clustering to segment customers for targeted marketing, dimensionality reduction to visualize high-dimensional data, and anomaly detection to identify fraud. Understanding these algorithms gives you a complete ML toolkit.

Key Insight

Unsupervised learning has no ground truth to evaluate against, making it fundamentally harder to validate than supervised learning. You must rely on domain knowledge, visual inspection, and internal metrics like silhouette scores. There is no single 'accuracy' number.

Key Concepts

1. K-Means Clustering

K-Means partitions data into K clusters by iteratively assigning points to the nearest centroid, then recomputing centroids. It is fast, scalable, and works well when clusters are roughly spherical and similarly sized. The main challenge: choosing K.

python 
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score
import matplotlib.pyplot as plt
import numpy as np

# Generate sample customer data
np.random.seed(42)
X = np.vstack([
    np.random.randn(100, 2) + [2, 2],
    np.random.randn(100, 2) + [-2, -2],
    np.random.randn(100, 2) + [2, -2],
])

# Elbow method to find optimal K
inertias = []
silhouettes = []
K_range = range(2, 11)

for k in K_range:
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X)
    inertias.append(km.inertia_)
    silhouettes.append(silhouette_score(X, km.labels_))

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))
ax1.plot(K_range, inertias, "bo-")
ax1.set_title("Elbow Method")
ax2.plot(K_range, silhouettes, "ro-")
ax2.set_title("Silhouette Score")
plt.show()

# Fit with best K and visualize
km = KMeans(n_clusters=3, random_state=42)
labels = km.fit_predict(X)
plt.scatter(X[:, 0], X[:, 1], c=labels, cmap="viridis", alpha=0.6)
plt.scatter(km.cluster_centers_[:, 0], km.cluster_centers_[:, 1],
            c="red", marker="x", s=200, linewidths=3)
plt.title("K-Means Clustering (K=3)")
plt.show()

2. Hierarchical and DBSCAN Clustering

Not all clusters are spherical. DBSCAN (Density-Based Spatial Clustering) finds clusters of arbitrary shape based on point density and automatically identifies noise points. Hierarchical clustering builds a tree (dendrogram) of nested clusters, giving you a multi-scale view of your data structure.

python 
from sklearn.cluster import DBSCAN, AgglomerativeClustering
from scipy.cluster.hierarchy import dendrogram, linkage

# DBSCAN — finds clusters of arbitrary shape
db = DBSCAN(eps=0.5, min_samples=5)
db_labels = db.fit_predict(X)
n_clusters = len(set(db_labels)) - (1 if -1 in db_labels else 0)
n_noise = (db_labels == -1).sum()
print(f"DBSCAN found {n_clusters} clusters, {n_noise} noise points")

# Hierarchical clustering with dendrogram
Z = linkage(X[:50], method="ward")  # Subset for readability
plt.figure(figsize=(12, 5))
dendrogram(Z, truncate_mode="level", p=5)
plt.title("Hierarchical Clustering Dendrogram")
plt.show()

3. Principal Component Analysis (PCA)

PCA is the most important dimensionality reduction technique. It finds the directions of maximum variance in your data and projects it onto those directions. This is essential for visualization (projecting high-dimensional data to 2D), noise reduction, and speeding up subsequent algorithms.

python 
from sklearn.decomposition import PCA
from sklearn.datasets import load_digits

# Load high-dimensional data (64 features per image)
digits = load_digits()
X_digits = digits.data  # Shape: (1797, 64)

# Fit PCA and examine explained variance
pca = PCA()
pca.fit(X_digits)

# How many components to keep?
cumulative_var = np.cumsum(pca.explained_variance_ratio_)
n_95 = np.argmax(cumulative_var >= 0.95) + 1
print(f"Components for 95% variance: {n_95} out of {X_digits.shape[1]}")

# Reduce to 2D for visualization
pca_2d = PCA(n_components=2)
X_2d = pca_2d.fit_transform(X_digits)

plt.scatter(X_2d[:, 0], X_2d[:, 1], c=digits.target, cmap="tab10",
            alpha=0.5, s=10)
plt.colorbar(label="Digit")
plt.title("Digits Dataset Projected to 2D with PCA")
plt.show()

4. t-SNE and UMAP for Visualization

While PCA preserves global structure, t-SNE and UMAP excel at preserving local structure — nearby points in high-dimensional space stay nearby in the visualization. These are your go-to tools for visualizing clusters in high-dimensional data like images and text embeddings.

python 
from sklearn.manifold import TSNE

# t-SNE visualization of digits
tsne = TSNE(n_components=2, random_state=42, perplexity=30)
X_tsne = tsne.fit_transform(X_digits)

plt.figure(figsize=(10, 8))
scatter = plt.scatter(X_tsne[:, 0], X_tsne[:, 1], c=digits.target,
                      cmap="tab10", alpha=0.6, s=10)
plt.colorbar(scatter, label="Digit")
plt.title("t-SNE Visualization of Handwritten Digits")
plt.show()

# Note: t-SNE is for visualization ONLY — do not use distances
# between clusters as meaningful. Use PCA for preprocessing.
t-SNE Pitfalls

t-SNE cluster sizes and distances are NOT meaningful. The perplexity parameter drastically changes the visualization. Always try multiple perplexity values (5, 30, 50) and never draw conclusions from a single t-SNE plot. It is a visualization tool, not an analysis tool.

5. Anomaly Detection

Anomaly detection identifies data points that deviate significantly from the norm. This is critical for fraud detection, network intrusion detection, manufacturing quality control, and health monitoring. Common approaches include Isolation Forest, One-Class SVM, and statistical methods.

python 
from sklearn.ensemble import IsolationForest

# Generate normal data with some anomalies
np.random.seed(42)
X_normal = np.random.randn(300, 2)
X_anomaly = np.random.uniform(-6, 6, (20, 2))
X_all = np.vstack([X_normal, X_anomaly])

# Isolation Forest
iso = IsolationForest(contamination=0.06, random_state=42)
predictions = iso.fit_predict(X_all)  # 1=normal, -1=anomaly

n_detected = (predictions == -1).sum()
print(f"Detected {n_detected} anomalies out of {len(X_all)} points")

plt.scatter(X_all[predictions == 1, 0], X_all[predictions == 1, 1],
            c="blue", label="Normal", alpha=0.5)
plt.scatter(X_all[predictions == -1, 0], X_all[predictions == -1, 1],
            c="red", label="Anomaly", marker="x", s=100)
plt.legend()
plt.title("Isolation Forest Anomaly Detection")
plt.show()

Self-Check

Before moving on, honestly answer these questions. If any answer is "no," go back and practice.

Can you explain when to use K-Means vs DBSCAN?

K-Means: spherical clusters, known K, fast. DBSCAN: arbitrary shapes, automatic K, handles noise, but sensitive to eps and min_samples. Use DBSCAN when clusters have irregular shapes or you don't know K.

Do you understand what PCA components represent?

Each principal component is a linear combination of original features pointing in the direction of maximum variance. The first PC captures the most variance, the second captures the most remaining variance orthogonal to the first, and so on.

Can you evaluate an unsupervised model without labels?

Use internal metrics: silhouette score (clustering quality), inertia (within-cluster variance), explained variance ratio (PCA). Also use domain knowledge and visual inspection. There is no single 'accuracy' for unsupervised learning.
