CNNs and Computer Vision

Learn how convolutional neural networks see the world — from filters and feature maps to building image classifiers from scratch with PyTorch and leveraging pre-trained models through transfer learning.

Estimated: 4–5 days
6 Core Concepts
3 Assignments

Why This Matters

Computer vision is one of the most successful applications of deep learning. From autonomous vehicles recognizing pedestrians to medical imaging detecting tumors, CNNs have revolutionized how machines interpret visual data. Understanding convolutional layers, pooling operations, and modern architectures gives you the foundation to tackle any image-based ML problem.

More importantly, the concept of transfer learning — taking a model trained on millions of images and fine-tuning it for your specific task — means you rarely need to train from scratch. A pre-trained ResNet or EfficientNet can be adapted to classify your custom images with only hundreds of examples and minutes of training time.

Key Insight

CNNs exploit a fundamental property of images: spatial locality. Nearby pixels are more related to each other than distant pixels. By using small filters that slide across the image, CNNs can learn local patterns (edges, textures) that compose into higher-level features (shapes, objects) layer by layer.

Key Concepts

1. Convolutional Layers and Filters

A convolutional layer applies a set of learnable filters (also called kernels) to the input. Each filter slides across the image, computing a dot product at every position, producing a feature map. Early layers learn simple patterns like edges and corners; deeper layers learn complex compositions like eyes, wheels, or letters. 

  Input Image (3x32x32)        Filter (3x3)        Feature Map
  ┌────────────────┐         ┌─────────┐         ┌──────────────┐
  │                │   *     │ -1  0  1│    =    │              │
  │   RGB pixels   │         │ -2  0  2│         │  Edge-detect │
  │   (3 channels) │         │ -1  0  1│         │  activations │
  └────────────────┘         └─────────┘         └──────────────┘

  Conv2d(in_channels=3, out_channels=32, kernel_size=3, padding=1)
  Parameters: 32 filters * (3 * 3 * 3 + 1 bias) = 896
Figure 1: A convolutional filter slides across the input to produce a feature map
python 
import torch
import torch.nn as nn

# Understanding convolution dimensions
conv = nn.Conv2d(
    in_channels=3,       # RGB input
    out_channels=32,     # 32 learned filters
    kernel_size=3,       # 3x3 filter
    stride=1,            # Move 1 pixel at a time
    padding=1            # Preserve spatial dimensions
)

# Simulate a batch of 4 RGB images, 32x32 pixels
x = torch.randn(4, 3, 32, 32)  # (batch, channels, height, width)
output = conv(x)

print(f"Input shape:  {x.shape}")        # [4, 3, 32, 32]
print(f"Output shape: {output.shape}")   # [4, 32, 32, 32]
print(f"Parameters:   {sum(p.numel() for p in conv.parameters())}")

2. Pooling and Architecture Patterns

Pooling layers reduce spatial dimensions while retaining the most important information. Max pooling takes the maximum value in each window, effectively keeping the strongest activations. This makes the network more robust to small translations and reduces computation.

The standard CNN architecture pattern is: Conv → ReLU → Pool, repeated with increasing filter counts. As spatial dimensions shrink, channel count grows, creating a pyramid of increasingly abstract feature representations.

python 
# Common CNN building blocks
class ConvBlock(nn.Module):
    """Conv -> BatchNorm -> ReLU -> MaxPool"""

    def __init__(self, in_ch, out_ch, pool=True):
        super().__init__()
        layers = [
            nn.Conv2d(in_ch, out_ch, kernel_size=3, padding=1),
            nn.BatchNorm2d(out_ch),
            nn.ReLU(inplace=True),
        ]
        if pool:
            layers.append(nn.MaxPool2d(kernel_size=2, stride=2))
        self.block = nn.Sequential(*layers)

    def forward(self, x):
        return self.block(x)

# Example: input 3x32x32
block = ConvBlock(3, 32)
x = torch.randn(1, 3, 32, 32)
print(f"After ConvBlock: {block(x).shape}")  # [1, 32, 16, 16]
Pro Tip

Always use BatchNorm2d after convolutions. It normalizes activations, stabilizes training, and often allows you to use higher learning rates. The pattern Conv → BN → ReLU is the standard building block of modern CNNs.

3. Building a CNN from Scratch (PyTorch)

Let us build a complete CNN for CIFAR-10 classification (10 classes of 32x32 color images). The architecture follows the pyramid pattern: progressively more filters and smaller spatial dimensions, ending with a fully connected classifier.

python 
import torch
import torch.nn as nn
import torch.optim as optim
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

class SimpleCNN(nn.Module):
    """A simple CNN for CIFAR-10 classification."""

    def __init__(self, num_classes=10):
        super().__init__()

        # Feature extractor: Conv blocks with increasing channels
        self.features = nn.Sequential(
            # Block 1: 3 -> 32 channels, 32x32 -> 16x16
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),

            # Block 2: 32 -> 64 channels, 16x16 -> 8x8
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),

            # Block 3: 64 -> 128 channels, 8x8 -> 4x4
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.BatchNorm2d(128),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),
        )

        # Classifier
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(128 * 4 * 4, 256),
            nn.ReLU(inplace=True),
            nn.Dropout(0.5),
            nn.Linear(256, num_classes),
        )

    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

model = SimpleCNN()
total_params = sum(p.numel() for p in model.parameters())
print(f"Total parameters: {total_params:,}")
python 
# Training loop
def train_one_epoch(model, loader, criterion, optimizer, device):
    model.train()
    running_loss = 0.0
    correct = 0
    total = 0

    for images, labels in loader:
        images, labels = images.to(device), labels.to(device)

        optimizer.zero_grad()
        outputs = model(images)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        running_loss += loss.item() * images.size(0)
        _, predicted = outputs.max(1)
        total += labels.size(0)
        correct += predicted.eq(labels).sum().item()

    return running_loss / total, correct / total

# Setup
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = SimpleCNN().to(device)
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

# Train for 20 epochs
for epoch in range(20):
    loss, acc = train_one_epoch(model, train_loader, criterion, optimizer, device)
    print(f"Epoch {epoch+1:2d} | Loss: {loss:.4f} | Acc: {acc:.4f}")

4. Transfer Learning with Pre-trained Models

Instead of training from random weights, transfer learning starts with a model pre-trained on ImageNet (1.2 million images, 1000 classes). You replace the final classification layer and fine-tune on your dataset. This works because early CNN layers learn universal features (edges, textures) that transfer across domains.

python 
import torchvision.models as models

# Load a pre-trained ResNet-18
model = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1)

# Freeze all layers (use pre-trained features as-is)
for param in model.parameters():
    param.requires_grad = False

# Replace the final fully connected layer for our task
num_features = model.fc.in_features
model.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(num_features, 10)   # 10 classes for CIFAR-10
)

# Only the new fc layer has requires_grad=True
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
total = sum(p.numel() for p in model.parameters())
print(f"Trainable: {trainable:,} / {total:,} ({trainable/total:.1%})")
Input Size Matters

Pre-trained models expect specific input sizes and normalization. ResNet expects 224x224 images normalized with ImageNet mean [0.485, 0.456, 0.406] and std [0.229, 0.224, 0.225]. Always match these preprocessing steps when using pre-trained weights, or the learned features will be meaningless.

5. Data Augmentation Strategies

Data augmentation artificially expands your training set by applying random transformations to images: flipping, rotation, cropping, color jittering. This teaches the model invariance to these transformations and dramatically reduces overfitting, especially with small datasets.

python 
from torchvision import transforms

# Training transforms: aggressive augmentation
train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224, scale=(0.8, 1.0)),
    transforms.RandomHorizontalFlip(p=0.5),
    transforms.RandomRotation(15),
    transforms.ColorJitter(
        brightness=0.2, contrast=0.2,
        saturation=0.2, hue=0.1
    ),
    transforms.RandomAffine(degrees=0, translate=(0.1, 0.1)),
    transforms.ToTensor(),
    transforms.Normalize(
        mean=[0.485, 0.456, 0.406],
        std=[0.229, 0.224, 0.225]
    ),
])

# Validation transforms: NO augmentation, just resize and normalize
val_transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(
        mean=[0.485, 0.456, 0.406],
        std=[0.229, 0.224, 0.225]
    ),
])

6. Image Classification Pipeline

A complete image classification pipeline combines data loading, augmentation, model definition, training loop, and evaluation into a reproducible workflow. Here is the full pipeline putting all the pieces together.

python 
from torch.utils.data import DataLoader
from torchvision import datasets
import torch

# Load datasets with transforms
train_dataset = datasets.CIFAR10(
    root='./data', train=True,
    transform=train_transform, download=True
)
test_dataset = datasets.CIFAR10(
    root='./data', train=False,
    transform=val_transform, download=True
)

# DataLoaders
train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True, num_workers=4)
test_loader = DataLoader(test_dataset, batch_size=64, shuffle=False, num_workers=4)

# Evaluation function
def evaluate(model, loader, device):
    model.eval()
    correct = 0
    total = 0
    with torch.no_grad():
        for images, labels in loader:
            images, labels = images.to(device), labels.to(device)
            outputs = model(images)
            _, predicted = outputs.max(1)
            total += labels.size(0)
            correct += predicted.eq(labels).sum().item()
    return correct / total

test_acc = evaluate(model, test_loader, device)
print(f"Test Accuracy: {test_acc:.4f}")
How do I choose which pre-trained model to use?

ResNet-18/34: Good starting point, fast training, solid baseline. ResNet-50: More capacity, standard for most tasks. EfficientNet: Best accuracy-efficiency tradeoff. Vision Transformer (ViT): State-of-the-art but needs more data. Start with ResNet-18 and upgrade only if accuracy is insufficient.

Should I freeze or fine-tune all layers?

Start by freezing all layers except the final classifier (feature extraction). If accuracy plateaus, unfreeze the last few convolutional blocks and fine-tune with a very small learning rate (1e-5). This two-stage approach prevents catastrophic forgetting of pre-trained features.

Self-Check

Before moving to Chapter 07, honestly answer these questions.

Can you explain what a convolutional layer does with correct dimensions?

A Conv2d(in_channels, out_channels, kernel_size) applies out_channels filters of size kernel_size x kernel_size to the input. With padding=1 and stride=1, spatial dimensions are preserved. Each filter produces one feature map.

Do you understand when to use transfer learning vs training from scratch?

Use transfer learning when: you have limited data (less than 10K images), your domain is similar to ImageNet (natural images), or you need fast results. Train from scratch when: your data is very different from natural images (medical scans, satellite), or you have massive datasets (100K+).

Can you write a complete PyTorch training loop from memory?

The core loop: model.train(), zero_grad, forward pass, compute loss, backward, optimizer step. Evaluation: model.eval(), torch.no_grad() context, compute metrics. You should be able to write both without looking at documentation.
