Python for Data Science

Master the core Python data science stack — NumPy for numerical computation, Pandas for data manipulation, and Matplotlib for visualization. These three libraries form the foundation upon which every machine learning pipeline is built.

Why This Matters

Every machine learning model, from a simple linear regression to a billion-parameter transformer, begins its life as data in an array. Before you can train a model, you need to load data, clean it, transform it, and understand its shape. Python's data science stack — NumPy, Pandas, and Matplotlib — is how the entire industry does this.

A data scientist who can't fluently manipulate arrays and DataFrames is like a writer who can't type. The ideas might be brilliant, but the execution bottleneck will slow everything down. These libraries are not just tools; they are the language of data in the ML world.

Key Insight

NumPy operations execute in optimized C code, making them 10-100x faster than equivalent Python loops. Understanding vectorization is not premature optimization — it is the fundamental paradigm for working with data at any scale. If you find yourself writing a for loop over array elements, you are almost certainly doing it wrong.

Key Concepts

1. NumPy: The Foundation of Numerical Computing

NumPy (Numerical Python) provides the ndarray — a homogeneous, n-dimensional array that is the backbone of scientific computing in Python. Unlike Python lists, NumPy arrays store data in contiguous memory blocks with fixed types, enabling hardware-level optimizations that make vectorized operations dramatically faster.

python 
import numpy as np

# Creating arrays from different sources
a = np.array([1, 2, 3, 4, 5])                   # From a list
b = np.zeros((3, 4))                              # 3x4 matrix of zeros
c = np.random.randn(1000)                        # 1000 samples from N(0,1)
d = np.linspace(0, 2 * np.pi, 100)              # 100 evenly spaced points

# Shape and dtype are your two most important attributes
print(a.shape)   # (5,)
print(b.shape)   # (3, 4)
print(b.dtype)   # float64

# Vectorized operations — no loops needed
x = np.array([1.0, 2.0, 3.0])
y = np.array([4.0, 5.0, 6.0])

dot_product = np.dot(x, y)                        # 32.0
element_wise = x * y                               # [4.0, 10.0, 18.0]
normalized = (x - x.mean()) / x.std()             # Z-score normalization

Broadcasting is one of NumPy's most powerful features. It allows operations between arrays of different shapes by automatically expanding the smaller array. Understanding broadcasting eliminates the need for most explicit loops and reshaping:

python 
# Broadcasting: adding a vector to each row of a matrix
matrix = np.random.randn(5, 3)   # Shape: (5, 3)
bias = np.array([1, 2, 3])        # Shape: (3,)

result = matrix + bias              # Shape: (5, 3) — bias is broadcast

# Matrix multiplication — the core of neural networks
W = np.random.randn(3, 4)         # Weight matrix
X = np.random.randn(10, 3)        # 10 samples, 3 features each
output = X @ W                      # Shape: (10, 4) — matrix multiply

# Boolean indexing — selecting elements by condition
data = np.random.randn(1000)
outliers = data[np.abs(data) > 2.0]  # Elements beyond 2 std devs
print(f"Found {len(outliers)} outliers out of {len(data)} samples")
Common Pitfall: Views vs. Copies

Slicing a NumPy array returns a view, not a copy. Modifying the slice modifies the original. Use .copy() when you need an independent copy: safe_slice = data[0:5].copy(). This is one of the most common sources of subtle bugs in data preprocessing.

2. Pandas: Structured Data Manipulation

While NumPy excels at numerical arrays, real-world datasets have column names, mixed types, missing values, and timestamps. Pandas provides the DataFrame — a labeled, two-dimensional data structure that handles all of this. Think of it as a programmable spreadsheet with the performance of a database.

python 
import pandas as pd

# Loading data from CSV — the most common starting point
df = pd.read_csv("housing_prices.csv")

# First look: shape, types, and missing values
print(df.shape)                    # (20640, 10)
print(df.dtypes)                   # Column types
print(df.isnull().sum())           # Missing values per column
print(df.describe())               # Statistical summary

# Selecting columns and filtering rows
prices = df["median_house_value"]                 # Single column (Series)
subset = df[["latitude", "longitude", "price"]] # Multiple columns
expensive = df[df["median_house_value"] > 500000] # Boolean filter

# Method chaining — the Pythonic way to build data pipelines
result = (
    df
    .dropna(subset=["total_bedrooms"])
    .assign(rooms_per_household=lambda x: x["total_rooms"] / x["households"])
    .query("rooms_per_household < 10")
    .groupby("ocean_proximity")["median_house_value"]
    .mean()
    .sort_values(ascending=False)
)

Pandas has two critical operations you will use in nearly every ML project: handling missing data and groupby aggregation. Master these, and you can wrangle almost any tabular dataset.

python 
# Handling missing data — three strategies

# 1. Drop rows with any missing values (loses data)
df_clean = df.dropna()

# 2. Fill with a constant or statistic
df["total_bedrooms"].fillna(df["total_bedrooms"].median(), inplace=True)

# 3. Forward-fill for time series data
df["temperature"].fillna(method="ffill", inplace=True)

# GroupBy — split, apply, combine
sales_by_region = (
    df
    .groupby("region")
    .agg(
        total_sales=("sales", "sum"),
        avg_price=("price", "mean"),
        num_transactions=("transaction_id", "count")
    )
    .reset_index()
)

# Merging DataFrames — like SQL JOINs
customers = pd.read_csv("customers.csv")
orders = pd.read_csv("orders.csv")
merged = pd.merge(customers, orders, on="customer_id", how="left")

3. Data Visualization with Matplotlib and Seaborn

Visualization is not just for presentations — it is a debugging tool. Plotting your data before modeling reveals outliers, skewed distributions, correlations, and data quality issues that summary statistics alone will miss. A histogram can save you hours of debugging a model that simply had bad input data.

python 
import matplotlib.pyplot as plt
import seaborn as sns

# Set a clean style for all plots
sns.set_style("whitegrid")
plt.rcParams["figure.figsize"] = (10, 6)

# Distribution analysis — always start here
fig, axes = plt.subplots(1, 3, figsize=(15, 5))

# Histogram: shows the shape of a single variable
axes[0].hist(df["median_house_value"], bins=50, edgecolor="black")
axes[0].set_title("Price Distribution")
axes[0].set_xlabel("Price ($)")

# Scatter plot: reveals relationships between two variables
axes[1].scatter(df["median_income"], df["median_house_value"],
                alpha=0.1, s=5)
axes[1].set_title("Income vs Price")
axes[1].set_xlabel("Median Income")

# Box plot: shows spread and outliers per category
df.boxplot(column="median_house_value", by="ocean_proximity",
           ax=axes[2])
axes[2].set_title("Price by Location")

plt.tight_layout()
plt.savefig("eda_overview.png", dpi=150)
plt.show()
Pro Tip

Use sns.pairplot(df) to generate a grid of scatter plots for every pair of numerical columns. It is the fastest way to find correlations and unusual patterns in a new dataset. For large datasets, sample first: sns.pairplot(df.sample(1000)).

4. Data Cleaning and Preprocessing Pipelines

Real-world data is messy. You will encounter missing values, duplicate records, inconsistent formats, outliers, and incorrect data types. A robust data cleaning pipeline should be reproducible and documented — meaning you can run it again on new data without manual intervention.

python 
def clean_dataset(df):
    """Reproducible data cleaning pipeline."""
    df = df.copy()

    # 1. Remove exact duplicates
    n_before = len(df)
    df = df.drop_duplicates()
    print(f"Removed {n_before - len(df)} duplicate rows")

    # 2. Fix data types
    df["date"] = pd.to_datetime(df["date"], errors="coerce")
    df["price"] = pd.to_numeric(df["price"], errors="coerce")

    # 3. Handle missing values strategically
    numeric_cols = df.select_dtypes(include=[np.number]).columns
    categorical_cols = df.select_dtypes(include=["object"]).columns

    df[numeric_cols] = df[numeric_cols].fillna(df[numeric_cols].median())
    df[categorical_cols] = df[categorical_cols].fillna("Unknown")

    # 4. Remove statistical outliers (IQR method)
    for col in numeric_cols:
        Q1 = df[col].quantile(0.25)
        Q3 = df[col].quantile(0.75)
        IQR = Q3 - Q1
        mask = (df[col] >= Q1 - 1.5 * IQR) & (df[col] <= Q3 + 1.5 * IQR)
        df = df[mask]

    # 5. Standardize text columns
    for col in categorical_cols:
        df[col] = df[col].str.strip().str.lower()

    return df

df_cleaned = clean_dataset(df)

5. Feature Engineering with Pandas

Feature engineering is the art of creating new columns from existing data that make patterns more visible to machine learning models. A well-engineered feature can improve model performance more than any algorithm change. This is where domain knowledge meets data manipulation.

python 
# Feature engineering examples for housing data

# Ratio features — often more predictive than raw values
df["rooms_per_household"] = df["total_rooms"] / df["households"]
df["bedrooms_ratio"] = df["total_bedrooms"] / df["total_rooms"]
df["people_per_household"] = df["population"] / df["households"]

# Binning continuous variables
df["income_category"] = pd.cut(
    df["median_income"],
    bins=[0, 1.5, 3.0, 4.5, 6.0, np.inf],
    labels=["very_low", "low", "medium", "high", "very_high"]
)

# One-hot encoding for categorical features
df_encoded = pd.get_dummies(df, columns=["ocean_proximity"], drop_first=True)

# Log transform for skewed distributions
df["log_price"] = np.log1p(df["median_house_value"])

# Correlation analysis to validate features
correlations = df_encoded.corr()["median_house_value"].sort_values(ascending=False)
print("Top correlated features:")
print(correlations.head(10))

6. Performance: Vectorization and Memory Management

When working with datasets of millions of rows, performance matters. The difference between a vectorized operation and a Python loop can be the difference between processing in 2 seconds and 20 minutes. Here are the patterns that matter most.

python 
import time

# BAD: Python loop — O(n) with Python overhead per element
data = np.random.randn(1_000_000)
start = time.time()
result_slow = [x ** 2 + 2 * x + 1 for x in data]
print(f"Loop: {time.time() - start:.3f}s")  # ~0.8s

# GOOD: Vectorized NumPy — O(n) in optimized C
start = time.time()
result_fast = data ** 2 + 2 * data + 1
print(f"Vectorized: {time.time() - start:.3f}s")  # ~0.003s

# Memory optimization for large DataFrames
df = pd.read_csv("large_dataset.csv")
print(f"Memory before: {df.memory_usage(deep=True).sum() / 1e6:.1f} MB")

# Downcast numeric types to save memory
for col in df.select_dtypes(include=["int64"]).columns:
    df[col] = pd.to_numeric(df[col], downcast="integer")
for col in df.select_dtypes(include=["float64"]).columns:
    df[col] = pd.to_numeric(df[col], downcast="float")

# Use category type for low-cardinality string columns
df["country"] = df["country"].astype("category")

print(f"Memory after: {df.memory_usage(deep=True).sum() / 1e6:.1f} MB")
Pro Tip

For truly large datasets that don't fit in memory, use pd.read_csv("file.csv", chunksize=10000) to process data in chunks. Alternatively, consider Dask or Polars for DataFrames that scale beyond RAM.

Do you understand why vectorization is faster than Python loops?

The answer is not just "because C is faster." It is about memory locality, SIMD instructions, and eliminating Python interpreter overhead per element. NumPy arrays store data contiguously in memory, allowing the CPU to use cache efficiently and process multiple elements with a single instruction. If you can explain this to a colleague, you are ready.
