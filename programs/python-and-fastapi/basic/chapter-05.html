Chapter 05 · Phase 3: FastAPI Essentials

FastAPI Quickstart

Routes, path/query parameters, request/response models, and automatic documentation. Build your first production-quality API endpoint in minutes, not hours.

Estimated: 3–4 days
3 Advancement Levels
3 Assignments

Why This Matters

FastAPI is the fastest-growing Python web framework because it solves real problems: automatic request validation, interactive API documentation, native async support, and type-safe dependency injection. It is built on Starlette for the web layer and Pydantic for data validation — both battle-tested libraries.

In this chapter, you will build your first FastAPI application from scratch. By the end, you will have an API with properly typed endpoints, automatic OpenAPI documentation, and clean error handling — the foundation every subsequent chapter builds upon.

Key Insight

FastAPI turns your Python type hints into a fully-featured API. When you annotate a parameter as user_id: int, FastAPI automatically validates the input, converts it to an integer, generates API docs, and returns a helpful error message if validation fails. Type hints are not just documentation — they are runtime behavior.

Key Concepts

1. Your First FastAPI Application

A FastAPI application starts with a single file. The framework auto-discovers routes, generates OpenAPI documentation, and provides an interactive testing UI at /docs.

python 
from fastapi import FastAPI

app = FastAPI(title="My API", version="1.0.0")

@app.get("/")
async def root():
    return {"message": "Hello, World!"}

@app.get("/health")
async def health_check():
    return {"status": "healthy"}

# Run with: uvicorn main:app --reload

2. Path and Query Parameters

FastAPI extracts path parameters from the URL and query parameters from the query string. Type hints control validation and conversion.

python 
from fastapi import FastAPI, Query, Path

@app.get("/users/{user_id}")
async def get_user(
    user_id: int = Path(..., gt=0, description="The user ID"),
):
    return {"user_id": user_id}

@app.get("/users")
async def list_users(
    skip: int = Query(default=0, ge=0),
    limit: int = Query(default=20, le=100),
    q: str | None = None,
):
    return {"skip": skip, "limit": limit, "query": q}

3. Request Bodies and Response Models

FastAPI uses Pydantic models to define request bodies and response shapes. This gives you automatic validation, serialization, and documentation.

python 
from pydantic import BaseModel, EmailStr
from fastapi import FastAPI, HTTPException

class UserCreate(BaseModel):
    name: str
    email: str
    age: int | None = None

class UserResponse(BaseModel):
    id: int
    name: str
    email: str

@app.post("/users", response_model=UserResponse, status_code=201)
async def create_user(user: UserCreate):
    # user is already validated by Pydantic
    new_user = {"id": 1, **user.model_dump()}
    return new_user

4. Error Handling and Status Codes

FastAPI provides HTTPException for returning error responses with proper status codes. You can also define custom exception handlers for consistent error formatting.

python 
from fastapi import HTTPException
from fastapi.responses import JSONResponse

@app.get("/users/{user_id}")
async def get_user(user_id: int):
    user = find_user(user_id)
    if not user:
        raise HTTPException(
            status_code=404,
            detail=f"User {user_id} not found",
        )
    return user

# Custom exception handler
@app.exception_handler(ValueError)
async def value_error_handler(request, exc):
    return JSONResponse(
        status_code=422,
        content={"detail": str(exc)},
    )

5. Automatic API Documentation

FastAPI generates interactive API documentation automatically from your type hints and docstrings. Swagger UI is available at /docs and ReDoc at /redoc.

python

    title="User Management API",
    description="A complete API for managing users",
    version="1.0.0",
    docs_url="/docs",
    redoc_url="/redoc",
)

    "/users/{user_id}",
    response_model=UserResponse,

Self-Check

Before moving on, honestly answer these questions. If any answer is "no", go back and review.

Can you create a FastAPI endpoint with path parameters, query parameters, and a request body?

If you can write an endpoint that takes /items/{item_id}?skip=0&limit=10 with a JSON body — all type-validated — you understand FastAPI's core pattern. This combination covers 90% of real API endpoints.

Do you understand how response_model filters the output?

response_model controls which fields appear in the response. If your database model has a password_hash field but your response model does not, FastAPI automatically strips it from the output. This is critical for security.

Can you use the TestClient to write API tests?

FastAPI's TestClient (from Starlette) lets you call endpoints without starting a server. client.get('/users/1') returns a response you can assert against. This is how you test APIs in pytest.
