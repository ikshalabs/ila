Chapter 03 · Phase 2: Python Mastery

Python Standard Library

Master the batteries-included modules that make Python so productive: collections, itertools, pathlib, typing, contextlib, and functools. These tools eliminate entire categories of boilerplate code.

Estimated: 3–4 days
3 Advancement Levels
3 Assignments

Why This Matters

Python's standard library is famously described as "batteries included." Before you reach for a third-party package on PyPI, there is a good chance the standard library already solves your problem — with zero dependencies, guaranteed maintenance, and no supply chain risk. Mastering these modules is the difference between writing 50 lines of custom code and writing 3 lines that use a battle-tested standard library function.

In FastAPI applications specifically, you will use typing for every endpoint signature, pathlib for file handling, contextlib for database session management, functools for caching and decorators, and collections for data processing. These are not optional knowledge — they are daily tools.

Key Insight

The best Python developers know the standard library deeply. When you see someone import defaultdict, lru_cache, or contextmanager without hesitation, you are looking at someone who writes less code, introduces fewer bugs, and ships faster.

Key Concepts

1. The collections Module

The collections module provides specialized container types that solve common data structure problems more elegantly than basic lists and dicts.

python 
from collections import defaultdict, Counter, namedtuple, deque, OrderedDict

# ── defaultdict: no more KeyError checks ──
word_counts: defaultdict[str, int] = defaultdict(int)
for word in "the cat sat on the mat".split():
    word_counts[word] += 1
# defaultdict(int, {'the': 2, 'cat': 1, 'sat': 1, 'on': 1, 'mat': 1})

# Group items by category
items_by_type: defaultdict[str, list] = defaultdict(list)
for item in products:
    items_by_type[item["category"]].append(item)

# ── Counter: frequency analysis ──
status_codes = [200, 200, 404, 200, 500, 404, 200]
counts = Counter(status_codes)
print(counts.most_common(2))  # [(200, 4), (404, 2)]

# ── deque: efficient double-ended queue ──
recent_requests: deque[str] = deque(maxlen=100)
recent_requests.append("GET /api/users")    # O(1)
recent_requests.appendleft("POST /login")  # O(1)
# Automatically evicts oldest when full

2. The itertools Module

itertools provides memory-efficient tools for creating and manipulating iterators. These functions compose elegantly to build complex data pipelines without intermediate lists.

python 
from itertools import chain, islice, groupby, product, starmap
from itertools import accumulate, takewhile, dropwhile, zip_longest

# ── chain: flatten multiple iterables ──
admins = ["alice", "bob"]
editors = ["charlie", "diana"]
all_users = list(chain(admins, editors))

# ── islice: slice an iterator without loading all into memory ──
def read_logs():
    with open("server.log") as f:
        yield from f

first_100 = list(islice(read_logs(), 100))

# ── groupby: group sorted data ──
users = [
    {"name": "Alice", "role": "admin"},
    {"name": "Bob", "role": "admin"},
    {"name": "Charlie", "role": "editor"},
]
for role, group in groupby(users, key=lambda u: u["role"]):
    print(f"{role}: {list(group)}")

# ── product: cartesian product (great for test cases) ──
sizes = ["S", "M", "L"]
colors = ["red", "blue"]
variants = list(product(sizes, colors))
# [('S','red'), ('S','blue'), ('M','red'), ('M','blue'), ...]

3. pathlib — Modern File Paths

pathlib replaces os.path with an object-oriented API for filesystem paths. It is cross-platform, readable, and composable.

python 
from pathlib import Path

# ── Path construction ──
base = Path("data")
config_file = base / "config" / "settings.json"
# PosixPath('data/config/settings.json')

# ── Reading and writing ──
content = config_file.read_text(encoding="utf-8")
config_file.write_text('{"debug": true}')

# ── Directory operations ──
data_dir = Path("output")
data_dir.mkdir(parents=True, exist_ok=True)

# ── Glob patterns ──
python_files = list(Path("src").rglob("*.py"))

# ── Path properties ──
p = Path("/home/user/data/report.csv")
print(p.stem)      # "report"
print(p.suffix)    # ".csv"
print(p.parent)    # PosixPath('/home/user/data')
print(p.exists())  # True/False

4. typing — Type Hints for Everything

The typing module is the foundation of FastAPI's request validation and documentation generation. Every endpoint signature uses type hints to define parameter types and response shapes.

python 
from typing import (
    Optional, Union, Literal, TypeAlias,
    TypeVar, Generic, Callable, Annotated,
)

# ── Basic type hints ──
def greet(name: str, times: int = 1) -> str:
    return (name + "! ") * times

# ── Union and Optional (Python 3.10+: use | syntax) ──
def find_user(user_id: int) -> dict | None:
    ...

# ── Literal for constrained values ──
def set_status(status: Literal["active", "inactive", "banned"]):
    ...

# ── TypeAlias for complex types ──
JSON: TypeAlias = dict[str, "JSON"] | list["JSON"] | str | int | float | bool | None

# ── Generic types ──
T = TypeVar("T")

class Result(Generic[T]):
    def __init__(self, value: T | None = None, error: str | None = None):
        self.value = value
        self.error = error

    @property
    def is_ok(self) -> bool:
        return self.error is None

# ── Annotated for metadata (used by FastAPI) ──
UserId = Annotated[int, "Must be positive"]
Pro Tip

FastAPI uses Annotated extensively for dependency injection and parameter metadata. Annotated[int, Path(gt=0)] tells FastAPI that a path parameter must be a positive integer. Learning typing.Annotated now pays off enormously in Chapter 05.

5. contextlib — Context Managers

Context managers handle resource cleanup (database connections, file handles, locks). The contextlib module lets you create them without writing a full class.

python 
from contextlib import contextmanager, asynccontextmanager, suppress

# ── Generator-based context manager ──
@contextmanager
def database_session():
    """Yield a DB session, commit on success, rollback on error."""
    session = Session()
    try:
        yield session
        session.commit()
    except Exception:
        session.rollback()
        raise
    finally:
        session.close()

# Usage
with database_session() as db:
    db.add(user)

# ── suppress: ignore specific exceptions ──
with suppress(FileNotFoundError):
    Path("temp.txt").unlink()

# ── Async context manager (for FastAPI lifespan) ──
@asynccontextmanager
async def lifespan(app):
    # Startup
    db = await init_database()
    yield {"db": db}
    # Shutdown
    await db.close()

6. functools — Higher-Order Functions

functools provides tools for working with functions: caching, partial application, and reducing sequences. These are essential for performance optimization and creating reusable function patterns.

python 
from functools import lru_cache, cache, partial, reduce, wraps

# ── lru_cache: memoization ──
@lru_cache(maxsize=128)
def fibonacci(n: int) -> int:
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# Without cache: O(2^n). With cache: O(n). Massive difference.
fibonacci(100)  # instant

# ── cache (Python 3.9+): unlimited cache ──
@cache
def load_config() -> dict:
    return json.loads(Path("config.json").read_text())

# ── partial: fix some arguments ──
def power(base: int, exponent: int) -> int:
    return base ** exponent

square = partial(power, exponent=2)
cube = partial(power, exponent=3)
print(square(5))  # 25
print(cube(5))    # 125

# ── reduce: fold a sequence ──
numbers = [1, 2, 3, 4, 5]
total = reduce(lambda a, b: a + b, numbers)  # 15
Why lru_cache is so important for APIs

In a FastAPI application, you often have expensive computations or database lookups that return the same result for the same input. lru_cache stores results in memory so repeated calls with the same arguments return instantly. The maxsize parameter limits memory usage by evicting the least recently used entries. Use it for config loading, permission checks, and computed properties.

The functools.wraps pattern explained

@wraps(func) copies the original function's metadata (__name__, __doc__, __module__) to the wrapper function. Without it, decorated functions lose their identity, breaking debugging tools, FastAPI's automatic documentation, and introspection. Always use @wraps when writing decorators.

Self-Check

Before moving to Chapter 04, answer these honestly.

Can you use defaultdict, Counter, and deque from memory?

These three are the most commonly needed. defaultdict eliminates key-existence checks. Counter handles frequency analysis. deque provides O(1) append/pop from both ends and fixed-size sliding windows. If you need to look up the API each time, you need more practice.

Can you write a context manager using @contextmanager?

The pattern is: setup code, yield the resource, cleanup code in finally. If you can write a database session context manager from memory, you are ready for FastAPI's dependency injection, which uses the same pattern.

Do you understand when to use lru_cache vs. a regular dict cache?

lru_cache is for pure functions (same input always gives same output) with hashable arguments. Use a regular dict when you need manual invalidation, non-hashable keys, or async operations. lru_cache is thread-safe for reads but not for the computation itself.
