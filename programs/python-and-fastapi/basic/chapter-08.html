Chapter 08 · Phase 4: Production Backend

API Authentication and Security

OAuth2, JWT tokens, password hashing, CORS, and rate limiting. Secure your API against real-world threats with production-grade authentication.

Estimated: 3–4 days
3 Advancement Levels
3 Assignments

Why This Matters

An API without authentication is an API anyone can abuse. Security is not a feature you add at the end — it is a property that must be designed in from the start. In this chapter, you will implement OAuth2 with JWT tokens, the most common authentication pattern for modern APIs.

You will also learn about CORS (Cross-Origin Resource Sharing), rate limiting, and password hashing — the security fundamentals that every production API needs. Getting any of these wrong can lead to data breaches, denial of service, or unauthorized access.

Security is not optional

Never store passwords in plain text. Never trust client input. Never expose internal error details. Never disable CORS in production. These are not guidelines — they are non-negotiable rules that protect your users and your business.

Key Concepts

1. Password Hashing with bcrypt

Never store passwords in plain text. Use bcrypt (or argon2) to hash passwords with a salt. The hash is one-way — you can verify a password against the hash but cannot recover the original password.

python 
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

# Usage
hashed = hash_password("mysecretpassword")
print(verify_password("mysecretpassword", hashed))  # True

2. JWT Token Authentication

JSON Web Tokens (JWT) are self-contained tokens that encode user identity and permissions. They are stateless — the server does not need to store session data.

python 
from jose import JWTError, jwt
from datetime import datetime, timedelta

SECRET_KEY = "your-secret-key-here"
ALGORITHM = "HS256"
EXPIRE_MINUTES = 30

def create_access_token(data: dict) -> str:
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)

def decode_token(token: str) -> dict:
    try:
        return jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

3. OAuth2 with FastAPI Dependencies

FastAPI provides built-in OAuth2 support through its dependency injection system. The OAuth2PasswordBearer class handles token extraction from the Authorization header.

python 
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

async def get_current_user(
    token: str = Depends(oauth2_scheme)
) -> User:
    payload = decode_token(token)
    user = await user_repo.get_by_email(payload["sub"])
    if not user:
        raise HTTPException(status_code=401)
    return user

@app.get("/users/me")
async def read_current_user(
    user: User = Depends(get_current_user)
):
    return user

4. CORS and Rate Limiting

CORS controls which domains can call your API from a browser. Rate limiting prevents abuse by limiting how many requests a client can make in a given time window.

python 
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Rate limiting with slowapi
from slowapi import Limiter
from slowapi.util import get_remote_address

limiter = Limiter(key_func=get_remote_address)

@app.get("/api/data")
@limiter.limit("10/minute")
async def get_data(request: Request):
    return {"data": "sensitive"}

Self-Check

Before moving on, honestly answer these questions. If any answer is "no", go back and review.

Can you explain why bcrypt is preferred over SHA-256 for passwords?

SHA-256 is fast — which is bad for passwords because attackers can hash billions of guesses per second. bcrypt is intentionally slow and includes a salt, making brute-force attacks impractical. The work factor can be increased as hardware gets faster.

Do you understand the OAuth2 password flow?

The client sends username and password to the /token endpoint. The server verifies credentials, creates a JWT, and returns it. The client sends the JWT in the Authorization header for subsequent requests. The server validates the JWT without database lookups.

Can you configure CORS correctly for a frontend application?

Set allow_origins to your frontend's domain (never use * with credentials). Set allow_credentials=True if you use cookies. Set allowed methods and headers. Getting CORS wrong either blocks legitimate requests or opens security holes.
